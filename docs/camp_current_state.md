## 1. 현재 캠프 디자인 요약

### 핵심 콘셉트

라이트 유저용 엔드컨텐츠. 포켓몬을 캠프에 "배치"하면 알아서 살아가고, 매일 바뀌는 임무에 맞추면 조각을 더 받고, 조각을 모으면 이로치로 전환한다. 네코 아츠메(놓고 닫기) + 워들(같은 문제 공유) + 동숲(벌주지 않기) + 스타듀밸리(흔한 게 귀해지는 순간) 콘셉트 조합.

### 캠프 레벨 (20단계, 그룹 공동)

그룹 전체 누적 조각으로 캠프가 성장. 20명 활동 기준 약 1달에 만렙.

| 구간 | 레벨 | 구역 수 | 게이트 |
|------|------|--------|--------|
| 시작 | Lv.1~3 빈터~모닥불 | 3 | 기본 |
| 초중반 | Lv.4~6 텐트촌~작은정원 | 4 | 조각 색 다양화 시작 |
| 중반 | Lv.7~9 울타리집~작은마을 | 5 | |
| 후반 | Lv.10~14 광장~공원 | 6 | 에픽 이로치 가능(5색) |
| 엔드 | Lv.15~20 도서관~포켓몬왕국 | 7 | 전설/초전설 이로치 해금(7색) |

### 구역 시스템 (7개)

| 구역 | 타입 | 조각 | 기본해금 |
|------|------|------|---------|
| 풀숲 | 풀/벌레 | 초록 | O |
| 호수 | 물 | 파랑 | O |
| 모닥불 | 불 | 빨강 | O |
| 바위산 | 바위/땅 | 갈색 | X (3마리+) |
| 하늘 | 비행 | 하양 | X (2마리+) |
| 그림자숲 | 악/고스트 | 보라 | X (2마리+) |
| 전기탑 | 전기 | 노랑 | X (2마리+) |

구역 해금은 캠프 레벨에 의한 최대 구역 수 제한 + 해당 타입 배치 마리수 조건의 이중 게이팅.

### 조각 시스템

- **획득 경로**: 일상 소식(4시간마다, 자동), 참여형 이벤트(8시간마다, 버튼 탭), 임무 매칭 보너스
- **배수**: 타입 매칭 x1, 지정 포켓몬(IV미달) x2, 지정 포켓몬+IV충족 x5, 이로치 +1
- **유저당 일일 평균**: ~2~4조각

### 이로치 전환 비용

| 등급 | 조각 | 최소 색 | 기여도 | 배치 최소 |
|------|------|--------|--------|----------|
| 커먼 | 30 | 1 | - | 7일 |
| 레어 | 60 | 3 | - | 7일 |
| 에픽 | 120 | 5 | - | 7일 |
| 전설 | 250 | 7 | 50 | 7일 |
| 초전설 | 250 | 7 | 100 | 7일 |

기여도 = 유저 누적 획득 조각 합계 (소모해도 줄지 않음). 프리라이더 방지용.

### 배치 슬롯 (도감 수 기반)

| 도감 수 | 슬롯 |
|---------|------|
| 0~30 | 3 |
| 31~100 | 4 |
| 101~200 | 5 |
| 201+ | 6 |

### 매일 임무

- 자정(KST) 갱신, 구역당 1개 포켓몬 + IV 조건 (랜덤 스탯, 15~25)
- 연속 이틀 같은 포켓몬 배제
- 아침 공지 메시지 → [배치하기] 버튼

### 소식 & 이벤트

- **일상 소식**: 4시간마다, 배치된 포켓몬 1~2마리의 활동 텍스트 + 조각 자동지급, 30% 확률로 유저간 인터랙션
- **참여형 이벤트**: 8시간마다, 구역별 이벤트(물주기/도와주기 등), 30초 내 아무나 1명 반응, 주인 +3조각 / 도우미 +2조각
- 모든 캠프 메시지는 1시간 후 자동삭제

## 2. 이미 구현된 코드

### 파일 목록과 상태

| 파일 | 줄 수 | 상태 | 설명 |
|------|------|------|------|
| `config.py` (CAMP_ 상수) | ~150줄 | 완성 | 구역, 조각, 레벨20, 기여도, 뉴스/이벤트/인터랙션 템플릿 전부 정의 |
| `database/camp_queries.py` | 341줄 | 완성 | 배치 CRUD, 구역, 임무, 조각(지급/소모/로그), 이벤트, 레벨, 이로치 전환 전체 쿼리 |

| `handlers/camp.py` | 477줄 | 완성 | 그룹 `캠프` 커맨드, 구역선택/포켓몬선택/페이지네이션/배치해제/이벤트응답 콜백, 스케줄러잡 3개 |

| `assets/camp/` | 5파일 | 존재 | map_base.png, zones/{grass,lake,campfire}.png, camp_level1.png |

### main.py 등록 상태

- **try/except ImportError 패턴**으로 camp 코드 임포트 (없으면 `HAS_CAMP=False`로 스킵)
- DM 핸들러: `내캠프`, `이로치전환` MessageHandler 등록
- 그룹 핸들러: `캠프` MessageHandler 등록
- 콜백: `camp_*` (그룹), `cdm_*` (DM) CallbackQueryHandler 등록
- 스케줄러: `camp_mission` (매일 00:01 KST), `camp_news` (4시간 반복, 첫 실행 2시간후), `camp_event` (8시간 반복, 첫 실행 3시간후)

### DB 스키마 상태

**schema.py에 CAMP 테이블 정의 없음.** 기획서에는 7개 테이블이 명세되어 있고, camp_queries.py는 이 테이블들을 전제로 쿼리가 작성되어 있지만, `database/schema.py`에는 아직 마이그레이션이 추가되지 않았다. 운영 DB에도 테이블이 생성되지 않은 상태.

### git 상태

캠프 관련 파일들은 **전부 untracked** (커밋되지 않음):
- `database/camp_queries.py`
- `handlers/camp.py`
- `handlers/dm_camp.py`
- `services/camp_service.py`
- `utils/camp_card_generator.py`
- `assets/camp/*`

config.py와 main.py의 캠프 관련 변경사항도 아직 커밋되지 않은 것으로 추정 (MEMORY.md에 "미커밋 파일" 언급).

### 완전히 빠진 것

- **캠프 카드 이미지 실제 사용**: camp_card_generator.py가 준비되어 있으나 어디서도 호출되지 않음. 소식/이벤트 메시지에 이미지 첨부 로직 필요.
- **이벤트 타임아웃 처리**: 30초 지나면 이벤트가 어떻게 되는지 명시적 코드 없음. `schedule_delete`로 메시지는 삭제되지만, 30초 후 "시간 초과" 메시지를 보내는 로직은 없음.
- **크로스채팅방 배치 충돌**: 같은 포켓몬을 여러 채팅방에 배치할 때의 정책이 UNIQUE(user_id, pokemon_id)로 자동 이동됨 (ON CONFLICT DO UPDATE). 의도적인지 확인 필요.
- **배치 시 pokemon_id vs instance_id 혼동 위험**: camp_placements 테이블은 pokemon_id(도감번호)를 저장하는데, 같은 종을 여러 마리 소유한 경우 어떤 개체를 배치했는지 구분 불가. camp_queries.py의 JOIN도 `up.pokemon_id = cp.pokemon_id`로 되어 있어 같은 종 여러 마리가 있으면 의도치 않은 결과 가능.

## 4. 유저 페인포인트 (캠프가 해결하려는 문제)

### 도감작 현타

라이트 유저는 포켓몬을 모으지만, 모은 포켓몬으로 할 수 있는 게 배틀밖에 없다. 배틀은 고인물 영역이고 덱짜기 진입장벽이 높다. 결국 "이걸 왜 모으지?" 현타가 오고 이탈.

### 이로치 파밍 경로 부재

v2.5 패치로 자연 스폰 이로치 확률이 10%에서 0%로 변경됨. 강스 2%, 아케이드 1%만 남음. Lv.4+ daily_shiny 확정은 있지만 제한적. 캠프의 조각→이로치 전환이 새로운 이로치 획득 경로가 됨.

### 라이트 유저 엔드컨텐츠 부재

현재 엔드컨텐츠 경로:
- 허슬형: ㅊ → 팀구성 → 배틀 → 토너먼트 → 랭크전 (있음)
- 라이트형: ㅊ → 도감작 → ... → 현타 (없음)

캠프가 라이트형의 엔드컨텐츠 역할.

### 채팅방 대화 소재 부족

"오늘 임무 뭐야?" "이상해꽃 있어?" 같은 자연스러운 대화 유도. 워들 효과(같은 문제를 공유하는 느낌).

### 뉴비 이탈

feature_proposals.md에 따르면 D+1 리텐션 ~45%. 캠프 자체가 뉴비 솔루션은 아니지만, 7일 여정과 함께 수집 동기를 강화하는 역할.

## 5. 기존 게임 시스템과 캠프 통합 포인트

### 스폰/포획 (`spawn_service.py`, `group.py`)

- **현재 연결**: 없음
- **가능한 통합**: 캠프 레벨이 높은 채팅방에서 스폰 확률 보정, 캠프에서 배치 중인 포켓몬의 같은 타입이 스폰되면 캠프 보너스

### 랭크전 (`ranked_service.py`)

- **현재 연결**: 없음

### 토너먼트 (`tournament_service.py`)

- **현재 연결**: 없음
- **가능한 통합**: 대회 보상에 캠프 조각 포함

### 합성 (`fusion_service.py`, `dm_fusion.py`)

- **현재 연결**: 없음
- **가능한 통합**: 합성 시 배치 자동 해제 필요 (유령 배치 방지)

### 거래소 (`market_service.py`, `dm_market.py`)

- **현재 연결**: 없음
- **가능한 통합**: 교환/판매 시 배치 자동 해제 필요, 임무 조건 충족 포켓몬이 거래소에서 가치 상승

### 교환 (`dm_trade.py`)

### 대시보드 (`dashboard/`)

## 6. 텔레그램 제약사항

### 텍스트/메시지

- **메시지 길이**: 최대 4096자. 캠프 소식은 1~2줄로 설계되어 문제 없음.
- **parse_mode**: 커스텀 이모지 사용 시 반드시 `parse_mode="HTML"`. 현재 캠프 코드는 기본 이모지만 사용 중.
- **메시지 수정**: `edit_message_text`로 인라인 버튼 메시지 수정 가능. 캠프 배치 UI가 이 방식 사용.

### 인라인 키보드

- **InlineKeyboardButton 라벨**: 커스텀 이모지 불가, 기본 이모지만. 현재 코드 준수 중.
- **callback_data 제한**: 64바이트. `camp_pk_{uid}_{zone}_{instance_id}` 형식으로 uid/zone/id가 길어지면 초과 가능. 현재는 문제 없으나 주의 필요.
- **query.answer() 팝업**: 커스텀 이모지 불가. 현재 코드 준수 중.

### 이미지 전송

- **사진 전송**: `send_photo()`로 BytesIO 전송 가능. camp_card_generator가 JPEG BytesIO 반환하므로 바로 사용 가능.
- **전송 속도**: 이미지 생성+전송은 텍스트보다 느림. 소식이 4시간마다이므로 문제 없지만, 모든 소식에 이미지를 붙이면 부하 고려 필요.
- **그룹 제한**: 봇이 메시지/이미지를 과도하게 보내면 Telegram rate limit 발생. 캠프 메시지 설계(하루 4~5개)가 이를 고려한 것.

### DM vs 그룹

- **배치는 그룹에서** (같이 하는 느낌), **조각 확인/이로치 전환은 DM에서** (개인 정보). 현재 코드가 이 분리를 준수.
- **DM 알림**: 텔레 유저는 DM을 잘 안 봄 (feature_proposals.md 진단). 그래서 캠프 핵심 경험은 그룹 채팅방에서 발생하도록 설계.

### 자동삭제

- `schedule_delete()` 유틸로 메시지 자동삭제. 캠프 메시지는 1시간(3600초) 후 삭제. 채팅방 오염 방지.

### 스케줄러

- python-telegram-bot의 `job_queue`로 `run_daily`, `run_repeating` 사용. 이미 등록 완료.

## 7. 유저의 비전 (대화 이력 + MEMORY 기반)

### SNG/포코피아 스타일 비주얼

캠프 카드 이미지 생성기가 이미 구현됨 (구역별 그라데이션 + 글로우 + 파티클 + 스프라이트). 유저는 텍스트만이 아니라 시각적으로 캠프가 "살아있는" 느낌을 원함. camp_card_generator.py가 이를 위한 기반이나 아직 실제로 사용되지 않음.

### 캠프가 레벨과 함께 성장하는 느낌

20단계 레벨 시스템이 이를 구현. 빈 터(Lv.1) → 텐트(Lv.2) → 마을(Lv.9) → 왕궁(Lv.19) → 포켓몬 왕국(Lv.20). 시각적으로도 레벨에 따라 캠프 이미지가 달라지면 좋겠다는 의도 (assets/camp/camp_level1.png 존재).

### 포켓몬 배치 + 조건부 파밍

"이상해꽃(힘 20+)" 같은 임무 조건이 핵심. 특정 포켓몬 + 특정 IV 조건 → 보상 차등. 흔한 포켓몬도 임무 조건에 맞으면 가치가 생김. 거래소에서 찾기도 함.

### 소셜 경험 우선

모든 설계가 "옆 사람과 함께"를 전제. 그룹 단위 구역 해금, 공동 레벨, 참여형 이벤트의 주인+도우미 구조, 유저간 포켓몬 인터랙션 소식. feature_proposals.md의 핵심 철학: "이 게임은 텔레그램 내 커뮤니티를 결속시키는 소셜 보조도구"

### 별도 커밋/배포로 분리

MEMORY.md에 명시: "캠프 시스템 코드는 별도 커밋/배포로 분리." main.py에서 try/except ImportError로 camp 코드가 없어도 봇이 정상 동작하도록 격리됨.

### 채팅방을 더럽히지 않기

하루 캠프 메시지 최대 4~5개 (아침 임무 1 + 소식 2~3 + 이벤트 1). 모두 자동삭제. 짧고 깔끔한 1~2줄 포맷. 유저 @멘션 안 함.
