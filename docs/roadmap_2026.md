## 3. Phase 1 (Q2 2026): 기반 정비

> **기간**: 2026년 4월 ~ 6월 (12주)
> **테마**: 경제 안정화, 재화 순환, 온보딩 강화
> **목표**: BP 순환율 <2.0, 캐주얼 D+7 리텐션 25%, 배틀 참여율 15%

### 3.1 방생/합성 시스템 (4월 1주~3주)

**우선순위: 1순위** | 개발 기간: 3주 | 난이도: ★★★☆☆

포켓몬을 **방생**(릴리스)하여 재화를 회수하거나, 2마리를 **합성**하여 새 포켓몬을 얻는 시스템.

| 기능 | 설명 | 보상 |
|------|------|------|
| 방생 | 보유 포켓몬 1마리 방생 | 하이퍼볼 1개 |
| 방생 (이로치) | 이로치 방생 | 하이퍼볼 3개 |
| 방생 (전설) | 전설급 방생 | 하이퍼볼 5개 + 50BP |
| 합성 | 동일 포켓몬 2마리 합성 | 같은 종 1마리 (IV 재설정, 부모 중 높은 IV 보정) |
| 교차합성 | 다른 포켓몬 2마리 합성 | 랜덤 포켓몬 1마리 (같은 등급 내, IV 랜덤) |

**경제 효과:**
- **재화 소모처** 추가: 중복 포켓몬 → 하이퍼볼 전환
- **인플레 완화**: 마스터볼 대신 하이퍼볼 순환 강화
- **수집형 동기**: 합성으로 더 좋은 IV 추구
- **예상 일일 방생량**: 활성 유저 80명 x 평균 2마리 = 160마리/일 → 하이퍼볼 160개 소모처 생성

### 3.2 볼 다양화 (4월 3주~5월 2주)

**우선순위: 1순위** | 개발 기간: 3주 | 난이도: ★★★☆☆

현재 포켓볼/하이퍼볼/마스터볼 3종에서 **7종**으로 확장.

| 볼 종류 | 가격 | 효과 | 획득 방법 |
|---------|------|------|----------|
| 포켓볼 | 기본 | 기본 포획률 x1.0 | 일일 20개 지급 + 충전(200BP/100개) |
| 하이퍼볼 | 20BP | 포획률 x3.0 | BP 상점, 방생 보상 |
| 마스터볼 | 200BP | 100% 포획 | BP 상점 (일 3개), 토너먼트 보상 |
| **울트라볼** | 50BP | 포획률 x5.0 | BP 상점 (일 5개) |
| **타이머볼** | 30BP | 포획률 x1.0~x8.0 (스폰 경과시간 비례) | BP 상점 |
| **다크볼** | 25BP | 심야(2-5시) 포획률 x6.0, 주간 x1.5 | BP 상점 |
| **프리미어볼** | 비매품 | 포획률 x2.0 + 포획 시 이로치 확률 x2 | 마스터볼 10개 구매 보너스 1개 |

**포획률 공식 업데이트:**

```
최종 포획률 = min(1.0, base_catch_rate × ball_multiplier × event_boost)

볼별 배수:
- 포켓볼:    1.0x
- 하이퍼볼:  3.0x
- 울트라볼:  5.0x
- 타이머볼:  1.0 + (elapsed_seconds / SPAWN_WINDOW) × 7.0  (최대 8.0x)
- 다크볼:    6.0x (심야) / 1.5x (주간)
- 프리미어볼: 2.0x + 이로치 확률 2배
- 마스터볼:  100% (무조건 포획)
```

**경제 효과:**
- BP 소모처 대폭 증가 (울트라볼 50BP x 일 5개 = 250BP/일)
- 올빼미형에게 다크볼이라는 전용 보상 부여
- 캐주얼에게 타이머볼로 "기다리면 잡을 수 있다"는 직관적 재미
- 프리미어볼로 고래 유저의 마스터볼 대량 구매에 보너스 제공

### 3.3 경제 리밸런스 (5월 2주~3주)

**우선순위: 1순위** | 개발 기간: 2주 | 난이도: ★★☆☆☆

| 변경 항목 | 현재 | 변경 후 | 근거 |
|----------|------|---------|------|
| 마스터볼 일일 구매 | 3개/200BP | 2개/250BP | 인플레 방지 |
| 배틀 승리 BP | 20~30 | 15~25 | BP 유입 축소 |
| 토너먼트 우승 | 2마볼+200BP | 1마볼+150BP+칭호 | 마볼 유입 축소 |
| 교환 수수료 | 150BP | 100BP | 교환 활성화 |
| 합성 비용 | 없음(신규) | 50BP | BP 소모처 |
| 교차합성 비용 | 없음(신규) | 100BP | BP 소모처 |
| 방생 보상 | 없음(신규) | 하이퍼볼 1~5개 | 재화 순환 |
| 울트라볼 | 없음(신규) | 50BP/개, 일 5개 | BP 소모처 |

**예상 일일 BP 흐름 (활성 유저 80명 기준):**

```
[BP 유입]                              [BP 유출]
  배틀 승리: 30명 × 20BP = 600        마스터볼: 60명 × 500BP = 30,000
  배틀 패배: 30명 × 5BP = 150         울트라볼: 40명 × 150BP = 6,000  (신규)
  토너먼트:  150BP (우승+준우승)        합성:    30명 × 75BP = 2,250   (신규)
  포켓볼충전: 200BP × 0회 = 0          타이머볼: 20명 × 60BP = 1,200  (신규)
  ────────────────────                다크볼:  10명 × 50BP = 500    (신규)
  일일 유입: ~900 BP                   교환:    10명 × 100BP = 1,000
                                      ─────────────────────────
                                      일일 유출: ~40,950 BP
```

> 유출이 유입보다 훨씬 많은 것은 **축적된 BP 재고를 소화**하는 디플레 단계.
> BP 재고가 안정화되면 배틀 BP를 점진적 상향 가능.

## 6. Feature Deep-dive: 방생/합성 시스템

### 6.1 방생 (Release / 방생)

**개요:** 불필요한 포켓몬을 야생으로 돌려보내고 보상을 받는 시스템.

**명령어:**
```
방생 [번호]        — 내포켓몬 번호의 포켓몬 방생
방생확인           — 방생 전 확인 (실수 방지)
```

**방생 보상 테이블:**

| 포켓몬 등급 | 기본 보상 | 이로치 추가 | IV S급 추가 |
|------------|----------|-----------|------------|
| Common | 하이퍼볼 1개 | +하이퍼볼 2개 | +10BP |
| Rare | 하이퍼볼 2개 | +하이퍼볼 3개 | +20BP |
| Epic | 하이퍼볼 3개 + 20BP | +하이퍼볼 5개 | +30BP |
| Legendary | 하이퍼볼 5개 + 50BP | +하이퍼볼 10개 | +50BP |

**방생 제한:**
- 파트너 포켓몬 방생 불가
- 배틀팀 포켓몬 방생 불가
- 즐겨찾기(is_favorite) 포켓몬 방생 불가
- 일일 방생 한도: 20마리 (어뷰징 방지)
- 방생 후 30초 쿨다운 (실수 연속 방생 방지)

**DB 변경:**
```sql
CREATE TABLE IF NOT EXISTS release_log (
    id SERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES users(user_id),
    pokemon_id INTEGER NOT NULL,
    pokemon_name TEXT NOT NULL,
    rarity TEXT NOT NULL,
    was_shiny INTEGER NOT NULL DEFAULT 0,
    iv_total INTEGER,
    reward_hyper INTEGER NOT NULL DEFAULT 0,
    reward_bp INTEGER NOT NULL DEFAULT 0,
    released_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 6.2 합성 (Merge / 합성)

**개요:** 2마리의 포켓몬을 합쳐 1마리의 새 포켓몬을 얻는 시스템.

**합성 유형:**

| 유형 | 재료 | 결과 | 비용 |
|------|------|------|------|
| 동종 합성 | 같은 종 2마리 | 같은 종 1마리 (IV 향상) | 50BP |
| 교차 합성 | 다른 종 2마리 | 랜덤 1마리 (같은 등급) | 100BP |

**동종 합성 IV 규칙:**
```python
# 결과 포켓몬의 각 IV 스탯:
for stat in ['hp', 'atk', 'def', 'spa', 'spdef', 'spd']:
    parent_max = max(parent1[stat], parent2[stat])
    parent_min = min(parent1[stat], parent2[stat])
    # 70% 확률로 높은 쪽 기반, 30% 확률로 낮은 쪽 기반
    base = parent_max if random.random() < 0.7 else parent_min
    # +-3 범위 랜덤 보정
    result[stat] = clamp(base + random.randint(-3, 3), 0, 31)
```

→ 합성을 반복하면 IV가 **점진적으로 개선**되지만, 확정적이지 않음
→ 완벽 개체 달성까지 상당한 투자 필요 = 엔드게임 콘텐츠

**교차 합성 규칙:**
- 두 재료 중 높은 등급의 풀에서 랜덤 포켓몬 선정
- IV는 완전 랜덤 (0~31)
- Epic 이상 교차합성 결과는 80% 같은 등급, 20% 한 단계 아래
- 전설급 교차합성 불가 (전설은 동종합성만 허용)

**합성 제한:**
- 일일 합성 한도: 10회
- 재료 포켓몬은 영구 소멸 (방생과 동일, 복구 불가)
- 파트너/배틀팀/즐겨찾기 포켓몬은 합성 재료로 사용 불가
- 합성 결과 포켓몬의 친밀도는 0으로 리셋

**명령어:**
```
합성 [번호1] [번호2]     — 두 포켓몬 합성
합성확인                 — 합성 전 상세 확인 (결과 예측)
```

## 7. Feature Deep-dive: 볼 다양화

### 7.1 전체 볼 라인업

| 볼 | 이모지 | 가격 | 일일 한도 | 포획 배수 | 특수 효과 |
|----|--------|------|----------|----------|----------|
| 포켓볼 | 🔴 | 기본 | 20개 (+충전) | x1.0 | 없음 |
| 하이퍼볼 | 🔵 | 20BP | 무제한 | x3.0 | 없음 |
| 울트라볼 | 🟡 | 50BP | 5개 | x5.0 | 없음 |
| 타이머볼 | ⏰ | 30BP | 10개 | x1.0~x8.0 | 시간 경과 비례 |
| 다크볼 | 🌑 | 25BP | 10개 | x6.0/x1.5 | 심야 특화 |
| 프리미어볼 | ⚪ | 비매품 | - | x2.0 | 이로치 확률 x2 |
| 마스터볼 | 🟣 | 250BP | 2개 | 100% | 확정 포획 |

### 7.2 타이머볼 상세

```
타이머볼 포획 배수 = 1.0 + (경과초 / 스폰윈도우초) × 7.0

예시 (60초 스폰 윈도우):
- 0초 경과: x1.0 (포켓볼과 동일)
- 15초 경과: x2.75
- 30초 경과: x4.5
- 45초 경과: x6.25
- 55초 경과: x7.4
- 60초 경과: x8.0 (최대)
```

**전략적 의미:**
- "기다릴수록 강해진다" → 마지막 10초에 던지는 것이 최적
- 하지만 다른 유저에게 먼저 잡힐 수 있음 → 리스크/리워드 판단
- 캐주얼 유저에게 직관적 ("기다리면 잡을 수 있다")

### 7.3 다크볼 상세

```
주간(5:00~2:00 KST): 포획 배수 x1.5
심야(2:00~5:00 KST): 포획 배수 x6.0
```

**전략적 의미:**
- 올빼미형 유저에게 전용 보상 (심야에 다크볼 = 울트라볼 이상)
- 일반 시간대에도 하이퍼볼(x3.0)보다 약간 낮아서 완전 사장되지 않음
- 다크볼 + 심야 보너스 스폰 = 올빼미형의 확실한 어드밴티지

### 7.4 프리미어볼 상세

**획득 방법:** 마스터볼 10개 구매 시 자동으로 프리미어볼 1개 증정 (누적 아님, 10개 일괄 구매 시에만)

**특수 효과:**
- 기본 포획률 x2.0
- 프리미어볼로 포획 성공 시 이로치 판정을 한 번 더 수행 (이로치 확률 실질 2배)
- 이로치를 의도적으로 노리는 수집형 유저에게 최고의 볼

### 7.5 명령어 매핑

| 입력 | 사용 볼 |
|------|---------|
| ㅊ | 포켓볼 |
| ㅎ | 하이퍼볼 |
| ㅁ | 마스터볼 |
| ㅇ | 울트라볼 (신규) |
| ㅌ | 타이머볼 (신규) |
| ㄷ | 다크볼 (신규) |
| ㅍ | 프리미어볼 (신규) |

### 7.6 경제 영향 분석

**일일 BP 소모 증가 예상 (활성 유저 80명 기준):**

| 볼 | 일 사용 예상 | BP 단가 | 일 BP 소모 |
|----|-------------|---------|-----------|
| 울트라볼 | 200개 (40명 x 5개) | 50BP | 10,000BP |
| 타이머볼 | 150개 (30명 x 5개) | 30BP | 4,500BP |
| 다크볼 | 50개 (10명 x 5개) | 25BP | 1,250BP |
| **신규 볼 총 소모** | | | **15,750BP/일** |

→ 기존 마스터볼(250BP x 2 x 60명 = 30,000BP) + 신규 볼(15,750BP) = **45,750BP/일** 소모
→ BP 인플레이션 효과적 억제

## 8. Feature Deep-dive: 메가진화

### 8.1 설계 철학

메가진화는 **배틀의 판도를 바꾸는 엔드게임 메커니즘**이다.

- **1인개발 현실성**: 새 이미지/스프라이트 불필요. 기존 카드에 "메가" 마크 + 스탯 보정만으로 구현
- **희소성**: 메가스톤은 토너먼트/시즌 보상으로만 획득 → 보유 자체가 위상
- **전략적 깊이**: "누구를 메가진화시킬 것인가"가 팀 구성의 핵심 변수

### 8.2 메가진화 조건

```
[메가진화 가능 조건]
1. 해당 포켓몬의 메가스톤 보유
2. 대상 포켓몬 친밀도 MAX (5, 이로치는 7)
3. 배틀 중 1회만 발동 가능 (팀 전체 기준)
```

### 9.2 "45종 웨이브" 설계 근거

**왜 45종인가:**
1. **카드 이미지 제작**: 1인 개발 기준 45종 = 약 3~5일 작업량
2. **시드 데이터**: 45종의 타입/스탯/진화트리/스킬 설정 = 약 2일
3. **테스트**: 밸런스 확인 + 스폰 풀 통합 = 약 1~2일
4. **총 작업**: 약 1주일 → 월 1웨이브가 현실적
5. **유저 소비 속도**: 하드코어 유저가 45종 완성하는 데 약 2~3주 → 다음 웨이브 전에 소진 안 됨

**콘텐츠 스트레칭 전략:**
```
[웨이브 출시일]
  → 7일 출시 이벤트 (출현률 부스트)
  → 7일 안정화 (일반 스폰 풀 합류)
  → 7일 밸런스 패치 (배틀 데이터 기반 조정)
  → 7일 빌드업 (다음 웨이브 티저, 미리보기)
  → [다음 웨이브]
```

### 9.3 스폰 풀 관리

세대가 늘어나면 스폰 풀이 비대해져서 특정 포켓몬을 잡기 어려워지는 문제가 발생한다.

**해결: 로테이션 스폰 풀**

| 스폰 유형 | 포켓몬 풀 | 비율 |
|----------|----------|------|
| 고정 풀 | 전 세대 전설 + 에픽 | 20% |
| 주간 로테이션 | 매주 변경 (특정 세대 or 타입) | 50% |
| 글로벌 풀 | 전체 Common + Rare | 30% |

```
예시: 7월 3주차
  고정: 뮤츠, 칠색조, 레쿠쟈 등 (전 전설)
  주간: "물 타입 위크" → 물 타입 출현률 3배
  글로벌: 전 세대 Common/Rare 랜덤
```

→ 주간 로테이션으로 매주 다른 경험 + 특정 포켓몬을 노릴 수 있는 기간

## 10. Feature Deep-dive: 길드/크루 시스템

### 10.1 설계 철학

텔레그램 문화의 핵심: **"우리 방이 최고"** 경쟁심.
이미 채팅방별 포획 경쟁이 자연 발생 중 → 공식 시스템으로 전환.

### 10.2 크루 구조

```sql
CREATE TABLE IF NOT EXISTS crews (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    emoji TEXT NOT NULL DEFAULT '🏴',
    leader_id BIGINT NOT NULL REFERENCES users(user_id),
    level INTEGER NOT NULL DEFAULT 1,
    total_cp INTEGER NOT NULL DEFAULT 0,
    weekly_cp INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS crew_members (
    crew_id INTEGER NOT NULL REFERENCES crews(id),
    user_id BIGINT NOT NULL REFERENCES users(user_id),
    role TEXT NOT NULL DEFAULT 'member'
        CHECK(role IN ('leader','officer','member')),
    joined_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (crew_id, user_id)
);

CREATE TABLE IF NOT EXISTS crew_activity_log (
    id SERIAL PRIMARY KEY,
    crew_id INTEGER NOT NULL REFERENCES crews(id),
    user_id BIGINT NOT NULL,
    activity_type TEXT NOT NULL,
    cp_earned INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 10.3 크루 레벨

| 레벨 | 필요 총 CP | 혜택 |
|------|-----------|------|
| Lv.1 | 0 | 기본 (최대 15명) |
| Lv.2 | 5,000 | 최대 20명, 크루 도감 |
| Lv.3 | 15,000 | 최대 25명, 크루 칭호 커스텀 |
| Lv.4 | 35,000 | 최대 30명, 크루 전용 이벤트 참여 |
| Lv.5 | 70,000 | 크루 보너스 (포획률 +5%) |

### 10.4 크루 대항전 (월간)

| 항목 | 사양 |
|------|------|
| 기간 | 매월 마지막 주 (7일) |
| 규칙 | 크루별 주간 CP 합산 대결 |
| 참가 조건 | 크루 Lv.2 이상, 크루원 10명 이상 |
| 보상 | 1위 크루 전원 마스터볼 2 + 전용 칭호 |

### 10.5 크루의 사회형 유저 효과

- **소속감**: "우리 크루를 위해 ㅊ 친다" → 캐치 동기 강화
- **교환 허브**: 크루 내 교환 수수료 50% 할인 (50BP)
- **멘토링**: 고레벨 유저가 뉴비를 크루로 초대 → 온보딩 자연 해결
- **경쟁**: 크루 랭킹 → "다른 크루를 이기자" → 전체 활동량 상승

### 12.2 로드맵 후 재화 체계 (목표)

```
[재화 유입]                           [재화 유출]
  배틀 승리 (15~25BP) ───────┐   ┌──── 마스터볼 (250BP x 2/일)
  배틀 패배 (5BP) ───────────┤   ├──── 울트라볼 (50BP x 5/일)
  토너먼트 (150BP) ──────────┤   ├──── 타이머볼 (30BP x 10/일)
  포켓볼 충전 (200BP) ───────┤   ├──── 다크볼 (25BP x 10/일)
  시즌 보상 ─────────────────┤   ├──── 합성 비용 (50~100BP)
                             │   ├──── 진화석 (100~150BP)
                             │   ├──── 교환 수수료 (100BP)
                             │   ├──── 크루 생성 (500BP)
                             │   ├──── 아케이드 이용권 (200BP)
                             │   └──── 메가스톤 (2,000BP/월1)
  ──────────────────────────
  목표: 유입/유출 비율 ≈ 0.8~1.2 (약간의 디플레가 건강)
```

### 12.3 재화 공급 스케줄

| 재화 | 일일 최대 유입 (1인) | 월간 최대 유입 (1인) |
|------|---------------------|---------------------|
| BP | 배틀 10회(200) + 토너먼트(150) = **350BP** | ~10,500BP |
| 마스터볼 | 상점(2) + 칭호버프(1) = **3개** | ~90개 |
| 하이퍼볼 | 방생(최대20개) + 상점(무제한) = **유동적** | 유동적 |
| 포켓볼 | 20개 + 충전(무제한, BP 비용) = **유동적** | 유동적 |

| 재화 | 일일 최대 유출 (1인) | 월간 최대 유출 (1인) |
|------|---------------------|---------------------|
| BP | 마볼(500) + 울트라(250) + 타이머(300) + 다크(250) + 합성(500) + 아케이드(600) + 교환(300) = **2,700BP** | ~81,000BP |

→ 일일 유입(350BP) < 일일 유출(2,700BP) → **강한 디플레 압력**
→ BP 재고가 충분한 초기에는 건강한 디플레, 재고 소진 후 배틀 BP 상향으로 균형 조정

### 12.4 인플레이션 방지 안전장치

| 장치 | 트리거 | 자동 조치 |
|------|--------|----------|
| BP 총량 경보 | 전체 BP > 50,000 | 관리자 알림 |
| 마스터볼 총량 경보 | 전체 마볼 > 200개 | 관리자 알림 |
| BP 순환율 모니터링 | earned/spent > 2.0 | 상점 가격 자동 할인 (소모 촉진) |
| 마스터볼 사용률 모니터링 | used_master/total > 15% | 캐치 난이도 상향 검토 |
