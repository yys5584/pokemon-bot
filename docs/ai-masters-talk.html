Chapter 01

나는 뭘 잘하는가

AI가 코딩을, 디자인을, 번역을 대체하고 있음.
쓸수록 느끼게 됨 — 그러면 나는 뭘 팔아야 하지? 무슨 스킬을 익혀야 하지?

AI를 쓰면 쓸수록 결국 이런 질문에 도달하게 됨.

나는 뭘 잘하는가?

남들과 다른 사고를 가지려면?

내가 만들고 싶은 게 뭐지?

이건 AI한테 물어봐서 나오는 답이 아님.
자기 자신한테 물어야 함.

AI가 대체하는 것과 못 하는 것

대체되는 것

코드 작성, 디자인, 번역, 데이터 분석, 문서 작성. 실행 영역은 빠르게 대체되고 있음.

대체 안 되는 것

"뭘 만들지 아는 눈". 시장 감각, 유저 이해, 판단력. 이건 각자의 경험에서만 나옴.

핵심

같은 도구를 줘도 결과가 갈리는 이유 — 도구가 아니라 그 도구를 쓰는 사람의 관점이 다르기 때문.

기술을 더 배우는 게 답이 아닐 수 있음.
자기가 뭘 알고, 뭘 잘하고, 뭘 만들고 싶은지.
이 질문에 답할 수 있는 사람이 AI 시대의 대체불가능한 자원이 됨.

📌

내 경우 — 구찌 세일즈, IT기획, 크립토 에이전시 창업, 웹3 게임채널 5년. 정규 개발 루트는 아님. 내 역할은 항상 기획과 판단이었음. AI를 만나고 나서야 그 경험이 "대체불가능한 자원"이 된다는 걸 알게 됐음.

Chapter 02

같은 도구, 다른 결과

같은 Claude Code로 3개를 만듦. 2개는 죽고 1개가 터짐.

다마고치 봇

혼자 키우는 게임. 남한테 안 보이니까 자랑도 경쟁도 없음. 동기 부재.

디펜스 게임

구현은 화려. 근데 "ㅊ" 한 번 누르는 것보다 진입장벽이 높으면 아무도 안 함.

포켓몬 봇 → 1,000명 가입, DAU 200+, 리텐션 67%

도구는 같았음. 달라진 건 "내가 뭘 아는지"를 알게 된 것.

처음 2번은 "뭘 만들 수 있는가"를 고민했음. 도구 중심 사고.
3번째는 "내가 뭘 아는가"를 고민했음. 자기 이해 중심 사고.

5년간 텔레그램에서 쌓은 도메인 지식 — 유저가 뭘 원하는지, 채널장이 뭘 고민하는지, 어떤 구조가 바이럴을 만드는지. 이걸 AI와 결합했더니 결과가 나왔음.

"뭘 만들 수 있는가" (도구 중심) → "내가 뭘 아는가" (자기 이해 중심)
이 전환이 핵심이었음.

Chapter 03

실전에서 통한 것들

자기 이해가 먼저. 그 다음에 도구를 쓰는 방법이 따라옴.
도메인 지식을 AI와 결합할 때 실제로 통했던 것들.

도메인 × AI가 작동한 순간

사례 — 유튜브 "가장 많이 본 장면"

리서치도 도메인 감각이 있어야 뽑아낼 수 있음

유튜브에 가장 많이 본 위치가 표시됨. 일반적으로는 "재밌는 부분이구나"로 넘김. 게임 기획자 관점에서는 거기에 그 게임이 터진 이유가 있음.

→ 수집형 게임 열광 구간 = "뭔가 터지는 순간". 이걸 채팅방에서 모두가 보게 설계함.
사례 — 텔레그램 채팅방 크롤링

타겟을 아니까 데이터에서 기회가 보임

코인 채팅방을 크롤링. 코인 얘기 안 하고 있었음. 하락장에 사람들이 남아서 수다만 떰. 채널장 최대 고민 = 볼륨 활성화.

→ 5년간 이 생태계에 있었으니까 보인 기회. 놀거리를 넣으면 채널장이 알아서 확산시켜줌. 마케팅비 ₩0.
사례 — 일간 KPI 리포트

데이터 → 판단 → 실행까지 하루 단위로 가능

매일 23:55 리포트. "ㅊ 한 번만 치고 이탈한 유저가 많음" — 이 숫자를 보고 원인을 판단하고 5분 만에 패치.

→ 도메인 감각이 있으니까 숫자를 보고 바로 판단이 됨. AI가 실행을 해주니까 판단 직후 배포 가능.

도구 활용 — 바로 써먹을 수 있는 것들

1

리서치 → 기획을 한 흐름에서

리서치 자료를 AI한테 통째로 넘기고, 같은 대화 안에서 기획서를 깎아나감. 대화를 끊으면 맥락이 날아감.

2

방향을 확정하지 말고 문제만 던질 것

답을 정해놓고 물어보면 AI가 거기에 갇힘.

X"튜토리얼을 DM으로 보내게 해줘"
O"DM 안 보는 유저가 많아. 자연스럽게 녹이고 싶어."

→ AI가 "포획할 때 툴팁을 같이 띄우자"고 제안. 내가 생각 못 한 답이 나옴.

3

스크린샷 한 장 > 프롬프트 100줄

AI는 이미지를 잘 읽음. 빨간색으로 표시하고 "이거 왜 이래?" — 이게 제일 빠름.

4

AI에게 영혼을 부여할 것

프로젝트 규칙, 실패 기록, 도메인 맥락을 문서에 담아두면 새 대화에서도 같은 맥락 유지. 실수를 기록하면 같은 실수를 반복하지 않음.

5

반복 프로세스는 자동화

자주 하는 리서치→기획서 프로세스를 커스텀 스킬로 만듦. 주제만 던지면 커뮤니티 크롤링 → 트렌드 분석 → 기획서 자동 생성.

6

커뮤니티의 가치

혼자서는 "내가 뭘 모르는지"를 모름. 트위터 글 100개보다 사람 한 명과 10분 대화가 나음.
사람과 직접 대화해야 새로운 질문이 생기고, 그 질문을 AI한테 던지면 답이 나옴.

7

빠른 피벗

AI 덕분에 이터레이션 비용이 낮음. 틀렸으면 엎고 다시 시작. 하루 단위로 방향을 검증하고 수정하는 속도 — 도메인 감각과 도구의 조합에서 나옴.

도구는 어디서나 구할 수 있음.
당신만의 관점을 가진 사람이 그 도구를 가장 잘 쓰게 됨.
