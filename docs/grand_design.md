### 3-2. 인플레이션 제어

#### 동적 보상 배율 (자동 조절)

```python
def dynamic_bp_multiplier(total_circulation: int, active_users: int) -> float:
    """유통량 기반 자동 보상 조절"""
    target_per_user = 1500
    target_total = target_per_user * active_users
    ratio = total_circulation / target_total

    if ratio < 0.8:   return 1.3   # 경제 위축 → 보상 증가
    elif ratio < 1.0:  return 1.0   # 건강
    elif ratio < 1.5:  return 0.8   # 약한 인플레
    elif ratio < 2.0:  return 0.6   # 강한 인플레
    else:              return 0.5   # 비상 브레이크
```

**현재**: 695,450 / (400 × 1,500) = 1.16 → 배율 0.8 (경미한 감소 시작)

#### 일일 수익 체감 (고인물 인플레 방지)

```
일일 BP 수익 구간별 효율:
  0~500 BP:    100% (기본)
  500~1,000:    70%
  1,000~2,000:  40%
  2,000+:       20%
```

**효과**: 1위 peterchat7 같은 유저(일 2,800+ BP 수익)의 실수익이 ~1,000 BP로 제한

#### 소비 인센티브 (벌금 대신 보상)

```
주간 소비 보너스 (월요일 00:00 리셋):
  500+ BP 소비 → 다음 주 +10% BP 수익
  1,500+ BP 소비 → 다음 주 +20%
  3,000+ BP 소비 → 다음 주 +30%
```

#### 일일 할인 로테이션


## 4. 길드전(캠프전) 상세 설계

### 4-1. 핵심 컨셉

> **캠프 = 길드. 채팅방이 곧 길드홀.**
> **주간 사이클로 캠프끼리 경쟁. 비동기 참여. 텔레그램 DM으로 공격.**

### 4-2. 주간 캠프전 사이클

```
월~금: 평상시 캠프 활동
  - 배치 정산 (기존 3시간 사이클)
  - 캠프전 포인트 축적 (배치, 배틀, 포획 등)
  - 상대 캠프 정찰 가능

토~일: 캠프전 대결
  - 상대 캠프 방어라인 공격 (DM 비동기)
  - 1인 일일 3회 공격 제한
  - 결과가 캠프전 점수에 반영

일 24:00 KST: 주간 정산
  - 리그 내 순위 결정
  - 보상 분배
  - 영토 변동
```

### 4-3. 캠프전 매칭 (리그제)

**리그 구분:**

| 리그 | 캠프 레벨 | 그룹 규모 | 시즌 |
|------|----------|----------|------|
| 🥉 브론즈 | Lv.1~3 | 4캠프/그룹 | 월간 |
| 🥈 실버 | Lv.4~6 | 4캠프/그룹 | 월간 |
| 🥇 골드 | Lv.7~8 | 4캠프/그룹 | 월간 |
| 💎 마스터 | Lv.9~10 | 4캠프/그룹 | 월간 |

**시즌 종료 시:**
- 그룹 내 1등 → 상위 리그 승격
- 그룹 내 꼴등 → 하위 리그 강등
- 나머지 → 잔류

**참고 (CoC CWL)**: 라운드 로빈이라 1패해도 시즌 전체가 망하지 않음. 승강제로 장기 경쟁 동기.

### 4-4. 캠프전 점수 기여

**모든 활동이 캠프전 점수에 기여 (다양한 기여 경로 = 캐주얼 유저 유지):**

| 활동 | 포인트 | 일일 상한 | 대상 |
|------|--------|----------|------|
| 캠프 배치 정산 | 배치 점수의 100% | 기존 제한 | 배치 유저 |
| 랭크 배틀 승리 | +3 | 10전 | 배틀러 |
| 일반 배틀 승리 | +1 | 10전 | 배틀러 |
| 포켓몬 포획 | +1 | 20회 | 수집가 |
| **캠프전 공격** (주말) | **+5~15** | **3회** | 전사 |

**핵심**: 개인 기여 상한이 있어 웨일 1명이 캐리 불가. 참여 인원이 중요.

### 4-5. 캠프전 대결 메카닉 (비동기 배틀)

```
[공격 흐름]

1. 상대 캠프의 "방어 라인업" 확인
   → 방어 라인업 = 상대 캠프에 배치된 포켓몬들

2. DM에서 /캠프전공격 명령
   → 상대 캠프 선택 → 공격 대상 선택 (중복 불가)
   → 자기 팀(3마리)으로 상대 방어 포켓몬과 자동 배틀

3. 결과: ★☆☆ ~ ★★★
   ★☆☆ = 1포인트 (간신히 승리 또는 일부 승리)
   ★★☆ = 2포인트 (깔끔한 승리)
   ★★★ = 3포인트 (완승, HP 50%+ 잔여)

4. 결과가 캠프전 점수에 합산
```

**방어 측 보너스 (리니지 성벽 개념):**
- 방어 포켓몬 HP +20% 보정
- 연속 방어 시 피로도 -5%/회 (PoGO CP 감소 참고)
- → 방어 유리하지만 영구 독점은 안 됨

### 4-6. 보상 체계

**주간 보상:**

| 순위 | 캠프 보상 | 개인 보상 (기여도 비례) |
|------|----------|----------------------|
| 🥇 1등 | 캠프 금고 1,000 BP + 영토 선택권 | 100~300 BP |
| 🥈 2등 | 캠프 금고 600 BP | 60~180 BP |
| 🥉 3등 | 캠프 금고 300 BP | 30~100 BP |
| 4등 | 캠프 금고 100 BP | 15~50 BP |

**시즌(월간) 보상:**
- 리그 승격: 승격 보상 (특별 칭호 + BP)
- MVP: 개인 최다 기여 → 시즌 한정 칭호
- 참여 보상: 최소 기여 달성 시 전원 보장

**핵심 (전 게임 공통 교훈)**: 져도 최소 보상이 있어야 다음에도 참여함.

### 5-2. 영토 점령/유지

- **캠프전 1등** → 빈 영토 1개 점령 OR 상대 영토 1개 빼앗기
- **영토 유지비** (주간, 캠프 금고에서 차감):

| 보유 영토 수 | 영토당 유지비 | 총 유지비 |
|-------------|-------------|----------|
| 1개 | 300 BP/주 | 300 |
| 2개 | 500 BP/주 | 1,000 |
| 3개 | 1,000 BP/주 | 3,000 |
| 4개+ | 2,000 BP/주 | 8,000+ |

**체감 설계**: 영토를 많이 가질수록 유지비가 급격히 증가 → 독점 방지

### 5-3. 대시보드 세계지도

- `tgpoke.com`에 세계지도 탭 추가
- 각 영토에 보유 캠프 이름/레벨 표시
- 캠프전 결과에 따라 실시간 변동
- 시즌별 히스토리 기록

### 6-2. 탐지 시스템

#### 봇 탐지 (반응속도 분석)

```sql
-- catch_attempts에 reaction_ms 컬럼 추가
ALTER TABLE catch_attempts ADD COLUMN reaction_ms INTEGER;

-- 봇 의심 유저 탐지 (주간 배치)
SELECT user_id, COUNT(*) as attempts,
       AVG(reaction_ms) as avg_ms,
       STDDEV(reaction_ms) as stddev_ms,
       COUNT(*) FILTER (WHERE reaction_ms < 500) as fast_count
FROM catch_attempts
WHERE attempted_at > NOW() - INTERVAL '7 days'
GROUP BY user_id
HAVING COUNT(*) >= 10
  AND (STDDEV(reaction_ms) < 100        -- 비인간적 일관성
    OR AVG(reaction_ms) < 300            -- 불가능한 속도
    OR COUNT(*) FILTER (WHERE reaction_ms < 500)::float / COUNT(*) > 0.5)
```

**탐지 임계값:**

| 신호 | 조건 | 확신도 |
|------|------|--------|
| 표준편차 < 100ms (15회+) | 로봇적 일관성 | 0.8 |
| 평균 < 300ms | 인간 불가능 | 0.7 |
| 50%+ 500ms 미만 | 자동화 의심 | 0.6 |
| 3개 모두 해당 | 확정적 | 0.95 |

#### 다계정 탐지 (교환 그래프)

```sql
-- 일방적 교환 관계 탐지 (A→B만 보내고 B→A 없음)
WITH trade_flows AS (
    SELECT from_user_id, to_user_id,
           COUNT(*) as trade_count
    FROM trades WHERE status = 'accepted'
      AND created_at > NOW() - INTERVAL '30 days'
    GROUP BY from_user_id, to_user_id
)
SELECT a.from_user_id, a.to_user_id, a.trade_count as a_to_b,
       COALESCE(b.trade_count, 0) as b_to_a
FROM trade_flows a
LEFT JOIN trade_flows b
    ON a.from_user_id = b.to_user_id AND a.to_user_id = b.from_user_id
WHERE a.trade_count >= 5 AND COALESCE(b.trade_count, 0) = 0
```

#### 마켓 가격 이상치

```sql
-- 같은 판매자-구매자 쌍의 반복 거래 탐지
SELECT seller_id, buyer_id,
       COUNT(*) as tx_count,
       SUM(price_bp) as total_bp
FROM market_listings
WHERE status = 'sold'
  AND sold_at > NOW() - INTERVAL '30 days'
GROUP BY seller_id, buyer_id
HAVING COUNT(*) >= 3 AND SUM(price_bp) > 1000
```

### 6-3. 대응 시스템 (단계적 제재)

```
Level 0 (정상):    제한 없음
Level 1 (경고):    DM 경고, 관리자 통보. 점수 > 0.5
Level 2 (제한):    교환 3회/일, 마켓 3건. 점수 > 0.7 또는 2차 경고
Level 3 (중제한):  교환/마켓 차단, 포획만 가능. 점수 > 0.85
Level 4 (임시밴):  전 기능 7일 차단. 점수 > 0.95
Level 5 (영구밴):  관리자 수동 결정만. 자동 불가
```

**핵심 원칙:**
- Level 5는 **절대** 자동화하지 않음
- 24시간 이내 연속 제재 금지 (오탐 방지)
- 구독자는 임계값 +0.1 상향 (오탐 비용 높음)

## 부록 A: 전 게임 공통 검증된 설계 원칙

1. **비동기 참여가 핵심**: 기한 내 자유 참여. 실시간 강제 X → 텔레그램 봇에 완벽 부합
2. **개인 기여 상한**: 웨일 1명 캐리 방지 → 참여 인원이 중요
3. **최소 보상 보장**: 져도 뭔가 받아야 다음에도 참여
4. **주기적 리셋**: 영구 독점 방지, 새로운 기회
5. **다양한 기여 방식**: 전투 외에도 기여 → 캐주얼 유지
6. **시각적 진행**: 점수판, 레이스, 영토 맵 가시화
7. **사회적 유대**: 길드원에게 피해주기 싫은 마음 → 자발적 참여
8. **준비 + 실행 분리**: 전략 수립 시간이 커뮤니케이션 촉진
