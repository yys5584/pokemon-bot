1. 왜 필요한가

허슬형: ㅊ → 팀구성 → 배틀 → 토너먼트 → 랭크전 (엔드컨텐츠 ✅) 라이트형: ㅊ → 도감작 → ... → 현타 ("이걸 왜 모으지?") (엔드컨텐츠 ❌)

핵심 문제: 라이트 유저는 포켓몬을 모으지만, 모은 포켓몬으로 할 수 있는 게 배틀밖에 없다.

해결: 모은 포켓몬이 살아 움직이는 공간을 만든다. 캠프가 있는 채팅방 = "도시" → 유저 유입 + 바이럴 엔진.

15. UX 요약

유저가 배워야 할 것:

1. 소식 뜨면 접수 타임에 배치한다 (30분)

2. 보너스 포켓몬 맞추면 점수 높다

3. 조각 모으면 이로치 된다

16. 구현 우선순위

Phase 1 — 코어 재설계

  • DB 테이블 재설계 (필드/슬롯/점수/라운드)
  • 필드 시스템 (6종 타입 그룹)
  • 소유자 필드 선택 + 변경 (쿨 3시간)
  • 슬롯 시스템 (기본/소유자/구독 + 500명당 보너스)
  • 라운드 구조 (3시간, 접수 30분 + 파밍 2:30)
  • 라운드 보너스 포켓몬 + 개체값 조건 랜덤
  • 점수 시스템 (1/2/4/4/7)
  • 엔빵 분배 + 캠프 레벨 캡
  • 필드 귀속 조각
  • 배치 제한 (도감 기반 일일 횟수)
  • 소유자 배치 모드 (자유/승인제, 쿨 3시간)
  • 승인 30분 미응답 → 자동 승인
  • 이로치 전환 (타입 조각 + 결정 + 무지개 + 쿨타임)
  • 분해 시스템 (이로치 → 결정/무지개)
  • 캠프 레벨 10단계
  • DM 현황 (내캠프, 이로치전환)
  • 대시보드 캠프 탭

Phase 2 — 안정화

  • 참여형 이벤트 (30초 반응, 도와주기)
  • 포켓몬 친밀도
  • 이로치 전환 축하 메시지
  • 교환/방생 시 배치 자동 해제

Phase 3 — 확장

  • Lv.11~20 확장
  • 프리미엄 구독 ($19.9) 슬롯 +2~3
  • 캠프 날씨 / 주간 번들 / 캠프 방문자
  • 계절 이벤트 / 캠프 방문
