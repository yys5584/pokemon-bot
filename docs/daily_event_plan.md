## 리서치 요약

### 채팅방 미니게임 — 뭐가 먹히나

- **디스코드 뽀삐봇**: 19만 서버, 가상재화 도박(슬롯/블랙잭)이 압도적 인기. "도파민 풀충전". 채팅방 경제 시스템이 핵심 리텐션. ([한국 디스코드 리스트](https://koreanbots.dev/bots/poppy), [DC인사이드](https://gall.dcinside.com/mgallery/board/view/?id=discord&no=24162))
- **CommunityOne Hype Engine**: 데일리 퀘스트 도입 서버에서 **채팅 4배, 신규 멤버 채팅 2배, 28일 리텐션 10배**. ([CommunityOne](https://blog.communityone.io/best-fun-discord-bots/))
- **Rumble Royale**: 수십 명 참가, 봇이 랜덤 전투 내레이션. 순수 운 — 아무나 이길 수 있음. 제로 스킬 배리어. ([EJAW](https://ejaw.net/discord-game-bots/))
- **크래시 게임**: 모바일 카지노 35%. 배수 올라가다 터짐 — 멈출 타이밍이 전부. 라운드 수초~1분. 채팅에서 "나 2.3배에 멈춤!" "ㅋㅋ 터졌다" → 대화 소재 폭발. ([CCN](https://www.ccn.com/analysis/crypto/crash-games-the-most-addictive-game-in-crypto-gambling/))
- **Pokemoon**: 채팅 활성화량 비례 보스 스폰 → 채팅 = 보상의 직접 선순환. ([top.gg](https://top.gg/bot/1227573606285053992))
- **Pokebest 레이드**: 30분마다 보스, 기여도 TOP 3 보상, 0기여 = 0보상 → 무임승차 방지. ([top.gg](https://top.gg/bot/857650250940547115))

### 실패 패턴 (회피 대상)

- **탭투언 피로**: Hamster Kombat 86% 이탈. 게임성 없는 반복 = 3개월 수명 ([aelf Blog](https://blog.aelf.com/posts/lessons-for-telegram-mini-app-tap-to-earn-games))
- **이벤트 피로**: 2025 가챠 최대 불만. 의무감 되면 끝 ([ResetEra](https://www.resetera.com/threads/anyone-else-trapped-in-service-mmo-gacha-game-hell-spending-2-hrs-a-day-doing-dailies-for-all-of-them-that-adds-up.332996/page-4))
- **보상 실망 → 분노 이탈**: 기대 높이고 보상 안 주면 역효과 (Catizen 사례)
- **개인 플레이 위주**: 채팅방에서 소셜 상호작용 없으면 활성화 효과 제로

### 핵심 인사이트

> **채팅방에서 함께 보고 반응하는 것 자체가 콘텐츠.** 게임의 깊이보다 "공유 경험"이 중요.
> 30초~2분 안에 끝나고, 결과에 한마디 하고 싶은 게임이 최적.

## 게임 철학 적용 체크

| 원칙 | 적용 |
|------|------|
| 채팅방이 무대 | 모든 이벤트는 그룹 채팅방에서 진행, 결과도 채팅방에 표시 |
| 단순함이 생명 | 참여 = 명령어 1개 또는 버튼 1번. 설명 불필요 |
| KOL 관점 | "이벤트 때문에 우리 방이 활발해진다" → 봇 유지 동기 |
| 안 해도 되는가 | 참가상 200BP(체크인 수준). 안 해도 ㅊ는 재밌음 |
| 옆그레이드 | 성장축 추가 ❌, 새 놀거리 ⭕ |
| 텔레그램 밖 X | 채팅방 내 완결 |

## 핵심 컨셉

**한 줄**: 매일 저녁 구독 채널에 랜덤 미니게임 스폰 → 채팅방에서 30초~2분 플레이 → 등수별 보상.

**포지션**: ㅊ(포획)의 이벤트 버전. 매일 다른 게임이 열리니까 "오늘 뭐야?" 기대감.

**왜 필요한지**:
- 현재 채팅방 활성화 = ㅊ뿐. 도감 채우면 동기 소멸
- 매일 저녁 이벤트 → 피크 시간 채팅 볼륨 폭발
- 구독 채널 한정 → 채널장 구독 가치 증가

## 이벤트 시스템 구조

### 스폰

- **시간**: 매일 20:30~21:00 KST 사이 랜덤 (피크 시간대 정중앙)
- **장소**: `channel_owner` 구독 + `채널등록`된 채널 중 랜덤 1개
- **알림**: 채팅방에 이벤트 카드 이미지 + 안내 메시지
- **DM 알림**: 최근 3시간 내 활동 유저에게 "[채널명]에서 이벤트!" + 채널 초대링크
- **지속 시간**: 이벤트별 다름 (30초~2분)

### 이벤트 채널 등록 (옵트인 방식)

**신규 기능 업데이트 시:**
- `channel_owner` 구독자 전원에게 DM 발송: "매일 이벤트 기능이 추가되었습니다! 참여하시겠습니까?"
- 참여 선택 시 → 이벤트 등록할 채널 초대링크 입력 안내
- 미참여 선택 시 → 스킵 (나중에 DM에서 `채널등록`으로 언제든 가능)

**등록 방식:**
- `채널등록` (DM) → 채널 초대링크 입력 → 등록 완료
- **자기 채널이 아니어도 OK** — 원하는 채널 링크를 자유롭게 지정
- 1구독 = 1채널
- 초대링크는 언제든 `채널수정`으로 변경 가능

**해제:**
- `채널해제` (DM) 또는 구독 만료 시 자동 해제

## 미니게임 종류

### 1. 포켓몬 퀴즈 (Pokemon Quiz)

> "포켓몬 박사 대결"

**유형 (매일 랜덤)** — 1세대(No.1~151) 한정:
- **실루엣 퀴즈**: 포켓몬 실루엣 이미지 → 이름 맞추기
- **타입 퀴즈**: "불꽃타입이면서 비행타입인 포켓몬은?"
- **도감 설명 퀴즈**: "꼬리의 불꽃이 꺼지면 생명이 다한다" → 누구?

**플레이**:
- **5문제 연속 출제**, 문제당 30초 (총 약 3분)
- 매 문제: 실루엣 이미지 출제 → 유저들이 정답 입력 → **선착순 5명만 정답 인정** → 다음 문제
- 5문제 종료 후 **일괄 정산**
- 틀린 답 무시, 6번째 이후 정답도 무시
- 5문제 종료 후 정답 수 합산으로 등수 표시 (동점 시 총 응답 시간 빠른 순)
- 정답 공개: 실루엣 → 컬러 이미지 전환 ("오늘의 포켓몬은 뭘까요?" 감성)
- 0문제 맞춰도 참가 보상(200 BP) 지급

**보상 구조** (문제당, 5문제 종료 후 일괄 정산):
- 1등 정답 (최초 정답자): **IV 선택리롤 2개**
- 2~5등 정답: **IV 선택리롤 1개**
- 6등~ 정답: **500 BP**
- 참가 보상 (0문제): 500 BP
- 최대: 5문제 전부 1등 = IV 선택리롤 10개

**채팅방 효과**: "아 이거 뭐지" "5명 마감됐다 ㅋㅋ" "아 늦었어!" → 5문제 내내 긴장감+아쉬움

### 2. 배틀로얄 (Battle Royale)

> "운빨 서바이벌"

**플레이**:
- 이벤트 시작 → `참가` 버튼 (30초 모집, 최대 30명)
- 모집 종료 → 봇이 라운드별 랜덤 전투 자동 진행
- 매 라운드 2명 매칭 → 승자 생존, 패자 탈락
- **참가자의 대표 포켓몬 이름 사용** (팀 1번 또는 가장 강한 포켓몬)
- **순수 랜덤** — 스탯/IV/등급 무관. 잉어킹이 뮤츠 이길 수 있음
- 내레이션: 포켓몬 기술 사용 연출 ("피카츄가 뮤츠에게 백만볼트! 뮤츠 쓰러졌다!")
- 마지막 생존자 우승

**채팅방 효과**: "ㅋㅋ 내 피카츄가 뮤츠 잡았다" "잉어킹 사기캐 ㅋㅋ" → 관전 재미 폭발

## 보상 구조 — DB 실데이터 기반

> 기준: 대회 참가상 500BP+마볼1, 배틀 1판 71BP, 주간 포획 인당 ~3,000BP, 일일 체크인 200BP

### 기본 보상 (모든 이벤트 공통)

| 등수 | 보상 |
|------|------|
| 1등 | 1,500 BP |
| 2~3등 | 800 BP |
| 4~10등 | 500 BP |
| 참가 | 200 BP |

- 월 누적: 1등 매일 ≈ 45K BP / 참가만 ≈ 6K BP → 포획 수입(~12K BP/월)과 균형
- 참가상 200BP = 일일 체크인과 동급 → 부담 없는 수준

### 주간 보너스 (선택적 확장)

- 주 5회 이상 참가 → 보너스 마스터볼 1개
- 해당 주 1등 횟수 TOP 3 → 주간 MVP 칭호 (임시)

## 별도: 폭탄돌리기 (상시 미니게임)

> 매일 이벤트와 별개. 채팅방에서 아무 때나 즐기는 상시 명령어.

### 컨셉

- **명령어**: `폭탄` (그룹 채팅에서)
- 시작한 유저 + 참가 버튼 누른 유저들로 게임 시작 (최소 3명, 최대 15명)
- 폭탄이 랜덤 참가자에게 → 제한 시간 안에 `패스` 입력
- 패스하면 다른 랜덤 참가자에게 이동
- 시간 내 패스 못 하면 탈락
- 매 라운드 제한 시간 줄어듦 (10초 → 8초 → 6초 → 5초)
- 마지막 생존자 우승

### 보상

- 가벼운 수준: 우승 300BP, 2등 100BP (상시 미니게임이므로 낮게)
- 일일 참가 횟수 제한: 5회

### 구현

- `handlers/group_bomb.py` — 폭탄돌리기 핸들러
- 이벤트 시스템과 독립
- 난이도: **M** (실시간 턴제 + 타이머 + 특정 유저 입력 감지)
- 매일 이벤트 이후 별도 배포

## 구현 범위

### DB

```sql
-- 이벤트 채널 등록
CREATE TABLE event_channels (
    chat_id BIGINT PRIMARY KEY,
    owner_id BIGINT NOT NULL,
    registered_at TIMESTAMPTZ DEFAULT NOW()
);

-- 일일 이벤트 기록
CREATE TABLE daily_events (
    id SERIAL PRIMARY KEY,
    event_date DATE NOT NULL,
    event_type VARCHAR(20) NOT NULL,  -- quiz, battle_royale
    chat_id BIGINT NOT NULL,
    pokemon_id INT,
    started_at TIMESTAMPTZ,
    ended_at TIMESTAMPTZ,
    status VARCHAR(10) DEFAULT 'pending'  -- pending, active, ended
);

-- 참가자 기록
CREATE TABLE event_participants (
    id SERIAL PRIMARY KEY,
    event_id INT REFERENCES daily_events(id),
    user_id BIGINT NOT NULL,
    rank INT,
    score NUMERIC,
    reward_bp INT DEFAULT 0,
    participated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 파일 구조

```
handlers/group_event.py      # 이벤트 명령어 핸들러, 참가 콜백

database/event_queries.py    # 이벤트 DB 쿼리
```
