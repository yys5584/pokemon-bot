# 랭크전 시스템

## 개요
LoL 스타일 티어/RP 기반 랭크전. MMR 매칭, 시즌제, 디케이.

## 핵심 파일
- `services/ranked_service.py` — MMR 계산, 매칭, 결과 처리, 리셋
- `handlers/battle_ranked.py` — 그룹 랭전 UI
- `handlers/ranked_auto.py` — DM 자동 매칭 ("랭전")
- `database/ranked_queries.py` — 시즌 레코드, 매칭 조회
- `jobs/ranked_jobs.py` — 시즌 리셋, 디케이 스케줄러

| 티어 | RP 범위 | 디비전 |
|------|---------|--------|
| 언랭 | — | 배치 미완료 |
| 브론즈 | 0~199 | II(0~99), I(100~199) |
| 실버 | 200~399 | II, I |
| 골드 | 400~599 | II, I |

## RP 계산

### 승리
```
기본 30 RP
+ 티어차 보정: (상대 티어 - 내 티어) × 5 (최대 ±10)
+ 연승 보너스: 연승 × 2 (최대 10)
× MMR 보정: 1.0 + clamp((내_MMR - 예상_MMR) / 400, -0.3, 0.5)
× 윈트레이딩 감소: pair_decay 배율
```

### 패배
```
골드 이상: 20 RP 손실
브론즈/실버: 15 RP 손실
신규 보호 (첫 10전): 손실 50% 감소
```

### 윈트레이딩 방지 (같은 상대)
| 횟수 | 배율 |
|------|------|
| 1~2회 | 100% |
| 3회 | 50% |
| 4회 | 25% |
| 5회+ | 15% |
| 일일 5회 초과 | 0% (RP 미획득) |

## 매칭 로직 (DM 자동)
1. MMR 기반 점진적 확장: [200, 300, 400] 범위
2. 각 범위에서 10명 후보 → 코스트/규칙 검증
3. 폴백: 인접 티어(±1) 대상 재매칭

### 매칭 제외
- 자신, 최근 3경기 상대
- 연속 방어 패배 3회 이상
- 팀 없음/미달/코스트 초과/시즌 룰 위반

## 시즌
- **기간**: 2주 (목요일 00:00 KST 시작)
- **배치**: 5전 필요
- **승급 보호**: 12시간 (랭전 시작 시 해제)

### 주간 리셋 (목요일 00:05 KST)
1. 이전 시즌 보상 분배
2. MMR 소프트 리셋: `new = 1200 + (old - 1200) × 0.75`
3. 모든 참여자 → 언랭 + 배치 미완료
4. 새 시즌 공지 DM

### 중간 리셋 (7일차 00:10 KST)
1. RP 조정: `new = old × 0.6`
2. 연승 카운트 초기화
3. 승급 보호 해제
