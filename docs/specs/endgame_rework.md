# 엔드게임 콘텐츠 리워크 기획서

> 제련소 + 메가스톤 + 주간보스를 연동되는 엔드게임 콘텐츠로 개선
> 작성일: 2026-04-04

---

## 1. 제련소 개선

### 현재 상태
- 10마리 + 200BP 투입 → 게이지 적립 → 대성공(이로치/메가제련권) or 소각보상
- **관리자 전용으로 잠김** (테스트 상태)
- 소각 보상이 빈약 (50%로 BP 30~80 + 하이퍼볼)
- 게이지 진행이 너무 느림 (일반 10마리 = 0.5%, 100%까지 200회)
- config에 설정이 중복 정의됨

### 1-1. 관리자 전용 해제
- admin gate 제거
- 대신 포켓몬 30마리 이상 보유 시에만 접근 가능 (소프트 게이트)

### 1-2. config 중복 정리
- 제련소 설정이 config.py에 두 번 정의됨 → 첫 블록 삭제

### 1-3. 소각 보상 (일반 보상) 리디자인
매 제련마다 "꽤 괜찮은" 보상이 나와야 함. 마스터볼/하이퍼볼은 하위템.

| 등급 | 확률 | 보상 |
|------|------|------|
| ⬜ 일반 | 35% | BP 100-200 + 조각 3-5 |
| 🟢 고급 | 30% | BP 200-300 + IV선택리롤 1 + 조각 3-5 |
| 🔵 희귀 | 20% | IV스톤 1 + 성격변경권 1 + BP 200-400 |
| 🟣 영웅 | 10% | IV스톤 1 + IV선택리롤 2 + 성격변경권 2 + BP 300-500 |
| 🟡 전설 | 5% | IV스톤 2 + 이로치소환권 1 + 성격변경권 3 + BP 500-1000 |

### 1-4. 대성공 시스템
게이지가 오를수록 **대성공 확률**이 올라가고, 대성공 중 **메가스톤 비율**도 점진적으로 상승.
80%에서 대성공이 떠도 이로치가 나올 수 있는 구조 (좋지만 아쉬운 = 도파민).

| 게이지 | 대성공% | 대성공 내 메가:이로치 |
|--------|---------|----------------------|
| 0-19% | 2% | 메가 10% / 이로치 90% |
| 20-39% | 5% | 메가 15% / 이로치 85% |
| 40-59% | 9% | 메가 25% / 이로치 75% |
| 60-79% | 15% | 메가 40% / 이로치 60% |
| 80-99% | 22% | 메가 55% / 이로치 45% |
| **100%** | **확정** | **메가 100%** |

**대성공 보너스 보상** (메인 보상에 추가):
- **이로치 대성공**: 이로치 포켓몬 + IV선택리롤 2 + 성격변경권 1 + BP 500
- **메가스톤 대성공**: 특정 메가스톤 + IV스톤 1 + 성격변경권 2 + BP 1000
  - 메가스톤 등급 확률: 일반 40% / 레어 30% / 에픽 18% / 전설 9% / 초전설 3%
  - 해당 등급의 `MEGA_EVOLUTION_POOL`에서 랜덤 1종 선택
  - 이미 보유 중인 스톤이면 같은 등급 내 미보유 우선 선택
  - 해당 등급 전부 보유 시 상위 등급으로 승급
  - 전체 45종 보유 시 BP 1000 + 조각 10

### 1-5. 게이지 수치 조정

| 등급 | 현재 | 변경 | 10마리 기여 |
|------|------|------|-------------|
| 일반 | 0.05 | 0.3 | 3% |
| 레어 | 0.15 | 0.8 | 8% |
| 에픽 | 0.40 | 2.0 | 20% |
| 전설 | 1.00 | 5.0 | 50% |
| 초전설 | 2.50 | 12.0 | 100%+ (즉시) |

- 이로치 배율: x10 (유지)
- 구독 배율: free 1.0x / basic 1.2x / channel 1.5x (유지)
- 일반만으로 ~33회, 혼합이면 15-25회 정도면 100% 도달

### 1-6. 자동선택 버튼
- "자동선택" 버튼 → 현재 필터 기준으로 가장 낮은 가치 10마리 자동 선택
- 포켓몬 10마리 일일이 탭하는 번거로움 해소

### 1-7. 연속 실패 보정 (Pity)
- 연속 실패 카운터 추가
- 게이지 획득 보너스: `x (1 + 실패횟수 x 0.05)`, 최대 2배
- 20연속 실패 시 게이지 획득량 2배

### 1-8. 기존 제련권(mega_stone_ticket) 폐지
- 제련권 아이템 삭제, 기존 보유분은 랜덤 메가스톤으로 자동 변환

## 2. 메가스톤 시스템 (신규)

### 현재 상태
- 47종 메가진화 데이터 완비 (`models/pokemon_mega_data.py`)
- 제련 보상으로 메가스톤 제련권 획득 가능하나 **사용 경로 없음**
- 배틀 시스템에 메가 스탯 미적용

### 2-1. DB 스키마
- `user_pokemon` 테이블에 `mega_key TEXT DEFAULT NULL` 컬럼 추가
- 새 테이블 `user_mega_stones`:
  - `id SERIAL PK`
  - `user_id BIGINT NOT NULL`
  - `mega_key TEXT NOT NULL` (예: "mega_6_x")
  - `obtained_at TIMESTAMPTZ DEFAULT NOW()`
  - `used BOOLEAN DEFAULT FALSE`
  - `UNIQUE(user_id, mega_key)` — 유저당 같은 스톤 1개

### 2-2. 메가스톤 획득 경로
| 경로 | 조건 |
|------|------|
| 제련소 대성공 | 게이지 구간별 확률 (1-4 참조) |
| 주간보스 | 1위 보상 + 참여자 중 랜덤 3명 |
| 이벤트/대회 | 향후 추가 |

### 2-3. 메가스톤 메뉴 (DM)
- 명령어: "메가스톤" / "메가" / "mega"
- 메인 메뉴:
  - 보유 스톤 목록 (등급별 정렬, 사용/미사용 표시)
  - "적용하기" 버튼
- 등급별 뱃지: 🟢일반 🔵레어 🟣에픽 🟡전설 🔴초전설

### 2-4. 메가스톤 적용 플로우
1. 미사용 스톤 선택
2. 해당 원본 포켓몬(pokemon_id 매칭) 중 내 보유 목록 표시
3. 조건 체크:
   - 우정도 5 이상
   - 메가진화 미적용
   - 활성 상태 (영혼 X)
4. before/after 스탯 비교 패널 표시
5. 확인 → `user_pokemon.mega_key` 설정 + 스톤 사용 처리 + BP 500 차감

### 2-5. 배틀 통합
- `_prepare_combatant`에서 `mega_key` 체크
- 메가: `MEGA_BASE_STATS[mega_key]`에서 스탯 + 타입 로드
- 스킬: 원본 포켓몬의 스킬 유지
- 랭크 코스트: 원본 등급 기준 (변경 없음)
- 보스전/던전/대회 모두 메가 스탯 적용

### 2-6. 메가스톤 풀 (45종)

| 등급 | 수량 | 예시 |
|------|------|------|
| 일반 | 3종 | 메가독침붕, 메가피죤투, 메가깜까미 |
| 레어 | 15종 | 메가후딘, 메가팬텀, 메가강철톤 등 |
| 에픽 | 22종 | 메가리자몽X/Y, 메가갸라도스, 메가보만다 등 |
| 전설 | 2종 | 메가라티아스, 메가라티오스 |
| 초전설 | 3종 | 메가뮤츠X, 메가뮤츠Y, 메가레쿠쟈 |

### 2-7. 새 파일
| 파일 | 역할 |
|------|------|
| `handlers/dm_mega.py` | UI/핸들러 (메인메뉴, 스톤목록, 적용) |
| `services/mega_service.py` | 비즈니스 로직 (드롭 롤, 적용, 조회) |
| `database/mega_queries.py` | DB CRUD |

## 3. 주간보스 리워크 (고스톱 시스템)

### 현재 상태
- 주간 보스 1체, 500K HP 고정
- 1일 1회 공격, 최대 60턴 배틀
- 딜 순위 → 주간 보상

### 3-1. 핵심 변경: 고스톱 메카닉
**평가 기준**: 한 주간 여러 번 도전 가능, **최고 딜량 1회**가 랭킹 기록
**전략 핵심**: 오래 살아남을수록 더 많이 때림 → **방어가 핵심**

6포켓몬 팀으로 진입 → 라운드마다 "고" or "스톱" 선택 (최대 6고)

| 라운드 | 보스 ATK 배율 | 보상 배율 |
|--------|---------------|-----------|
| 1 (초전) | 1.0x | 1x |
| 2 (1고) | 1.4x | 2x |
| 3 (2고) | 1.9x | 3x |
| 4 (3고) | 2.5x | 4x |
| 5 (4고) | 3.2x | 5x |
| 6 (5고) | 4.0x | 6x |
| 7 (6고) | 5.0x | 7x |

### 3-2. 라운드 진행
- 각 라운드: 내 팀 vs 보스 (기존 배틀 엔진 활용)
- 보스 ATK/SPA에 라운드별 배율 적용
- 내 포켓몬 HP는 라운드 간 **유지** (회복 없음)
- HP 0이 된 포켓몬은 **사망** → 영혼 상태
- 라운드별 딜량 누적 = 해당 도전의 총 딜 (랭킹 기록)

### 3-3. 고/스톱 선택
- **스톱**: 현재 보상 수령 + 살아남은 포켓몬 무사 귀환
- **전멸**: 보상 절반 수령 + 죽은 포켓몬 전부 영혼화
- **고 선택 화면**: 팀원 HP 상태바 + "다음 라운드 보스 ATK: x배" 경고

### 3-4. 영혼 시스템
- 보스전에서 HP 0이 된 포켓몬 → `status = 'soul'`
- 영혼 상태: 배틀/던전/보스/대회 참가 불가
- 도감에는 표시 (회색/반투명)
- **부활**: 향후 과금/광고 연동 (이번 구현에서는 상태 전환만)
- DB: `user_pokemon`에 `status TEXT DEFAULT 'active'` 컬럼 (active / soul)

### 3-5. 보상 구조
**라운드 기본 보상** (x보상배율):
- BP 50-100 + 조각 2-3

**주간 보상** (최고 딜 랭킹 기준):

| 순위 | 보상 |
|------|------|
| 1위 | 1000 BP + 마스터볼 3 + **메가스톤 1개** |
| 2~3위 | 600 BP + 마스터볼 2 |
| 4~10위 | 400 BP + 마스터볼 1 + IV선택리롤 2 |
| 11~20위 | 200 BP + IV선택리롤 1 |
| 21~30위 | 100 BP + 성격변경권 1 |

**메가스톤 추가 드롭**: 참여자 전체 중 **랜덤 3명**에게 메가스톤 1개씩 추가 지급

### 3-6. 동적 HP 스케일링
- 전주 참여자 수 기반: `HP = max(200,000, 200,000 + 참여자수 x 5,000)`
- 최대 2,000,000 HP
- 첫 주(이력 없음): 기본 200,000

### 3-7. 팀원별 딜/생존 분석
- 라운드별 각 포켓몬의 딜량 + 잔여 HP 표시
- 사망 포켓몬 표시
- 고/스톱 선택 화면에서 팀 상태 한눈에 확인

## 시스템 연동 흐름

```
포켓몬 수집
    ↓
제련소 (포켓몬 10마리 + 200BP)
    ↓
게이지 적립 + 소각 보상 (IV스톤, 성격변경권, IV리롤 등)
    ↓ (대성공)
메가스톤 드롭 (일반~초전설)  ←  주간보스 보상
    ↓
메가진화 적용 (우정도 5 + BP 500)
    ↓
강화된 팀으로 주간보스 도전 (고스톱)
    ↓
더 높은 딜 → 더 좋은 보상 → 메가스톤 추가 획득
```

## 구현 순서

1. **Phase 1** (제련소) — config 정리, admin 해제, 보상 리디자인, 게이지 조정, 대성공 시스템
2. **Phase 2** (메가스톤) — DB 스키마 → queries → service → handler → 배틀 통합
3. **Phase 3** (주간보스) — 고스톱 리워크, 영혼 시스템, HP 스케일링, 메가스톤 드롭
