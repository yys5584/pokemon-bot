# Race Condition 수정 (2026-04-02)

## 개요
빠른 연타 입력 시 명령어가 중복 실행되어 BP/출석 보상 다중 지급, 랭크전 무한 매칭 등 발생.

## 핵심 파일
- `handlers/_common.py` — `_is_duplicate_message()` 중복 메시지 필터
- `handlers/battle_ranked.py` — 랭크전 유저별 실행 락
- `handlers/group.py` — 출석/!돈 중복 방지

## 문제
1. 텔레그램에서 같은 명령어를 빠르게 여러 번 보내면 핸들러가 동시 실행
2. DB 쓰기 전에 조건 체크 → 체크 통과 → 동시에 여러 건 쓰기 (TOCTOU)
3. 랭크전: 매칭 중 다시 "랭전" 입력하면 중복 매칭

## 수정
- 유저별 asyncio.Lock으로 동시 실행 차단
- `_is_duplicate_message()`: 같은 유저의 같은 명령어를 1초 내 중복 무시
- 랭크전: 매칭 진행 중 플래그로 재진입 차단

## 설정값
- `config.RANKED_DAILY_CAP = 20` — 일일 랭크전 상한
