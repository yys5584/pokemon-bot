# specs/ — 최종 구현 문서

기능별 최종 구현 상태를 기록하는 폴더.

## 규칙
- 파일명: `{기능명}.md` (예: `tarot.md`, `ranked_battle.md`)
- 기획/연구/초안은 `docs/` 루트에 그대로 둠
- 구현 완료 후 최종 상태만 여기에 정리
- 수정 시 날짜와 변경 내용 하단에 추가

## 템플릿
```markdown
# {기능명}

## 개요
한 줄 설명

## 핵심 파일
- `handlers/xxx.py` — 역할
- `services/xxx.py` — 역할

## 동작 흐름
1. 유저가 X 입력
2. Y 처리
3. Z 결과 전송

## DB 테이블
- `table_name` — 용도, 주요 컬럼

## 설정값
- `config.XXX` — 설명, 기본값

## 변경 이력
| 날짜 | 내용 | 커밋 |
|------|------|------|
| 2026-04-02 | 초기 구현 | abc1234 |
```
