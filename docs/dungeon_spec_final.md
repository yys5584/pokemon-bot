# 포켓몬 던전 (로그라이크) 최종 설계서

> 버전: v1.0 (2026-03-18)
> 기반: 외부 리서치(뱀서/슬더스/서머너즈워/AFK아레나/포켓로그) + 내부 경제 분석

---

## 1. 컨셉 요약

**1마리 포켓몬으로 도전하는 로그라이크 던전.**

- 매 층 랜덤 적과 자동 배틀 → 승리 시 버프 3개 중 택1 (뱀서 스타일)
- 버프가 쌓일수록 포켓몬이 괴물로 성장 → 어디까지 갈 수 있나?
- 투자한 포켓몬(이로치 + S급 IV + 고친밀도)일수록 고층 도달
- 매일 던전 테마가 바뀜 → 다양한 포켓몬 사용 유도
- 입장권 필요 → 자원 관리 + 긴장감

### 해결하는 문제
| 현재 문제 | 던전이 해결 |
|----------|-----------|
| DAU 200 이하 하락 | 일일 무료 입장권 → 매일 접속 동기 |
| 캠프 참여율 0% | 이로치=층수, 조각 수요 폭증 |
| 엔드게임 할 것 없음 | 랭킹 경쟁, 마일스톤 칭호 |
| 커먼/레어 쓸모없음 | 코스트별 버프 차등 → 커먼 역전 가능 |
| IV/이로치 투자 동기 없음 | S급 이로치 = 10층+ 차이 |

## 2. 게임 흐름

```
DM: "던전"
  → 입장권 확인
  → 오늘의 던전 테마 표시
  → 포켓몬 선택 (인라인 버튼)
  → 던전 시작!

[매 층]
  → 적 등장 (테마 기반 랜덤)
  → 자동 배틀
  → 승리 시: 버프 선택 (코스트별 주기)
  → 패배 시: 런 종료, 보상 정산

[5층마다 보스] [10층마다 관장급]
  → 보스 클리어 시 트레이드오프 버프 선택
  → 관장 클리어 시 마일스톤 보상
```

## 3. 던전 테마 (일일 랜덤)

매일 자정(KST) 테마가 바뀜. 테마에 따라 적 타입 분포와 보너스 버프가 달라짐.

| 테마 | 적 타입 비중 | 유리한 포켓몬 | 보너스 버프 |
|------|------------|-------------|-----------|
| 🌿 고대 숲 | 풀/독/벌레 60%, 기타 40% | 불/비행/얼음 | 불꽃 계열 버프 등장률 ↑ |
| 🔥 화산 | 불/드래곤/격투 60% | 물/땅/바위 | 물 계열 버프 등장률 ↑ |
| 🌊 심해 | 물/얼음/비행 60% | 전기/풀 | 전기 계열 버프 등장률 ↑ |
| ⚡ 발전소 | 전기/강철/노말 60% | 땅/격투/불 | 땅 계열 버프 등장률 ↑ |
| 🪨 동굴 | 땅/바위/고스트 60% | 물/풀/얼음 | 얼음 계열 버프 등장률 ↑ |
| 🔮 신전 | 에스퍼/악/페어리 60% | 악/강철/독 | 악 계열 버프 등장률 ↑ |
| 🐉 용의 둥지 | 드래곤 80% | 페어리/얼음 | 주말 특수 (토~일) |

- 테마에 유리한 타입으로 가면 상성 이점
- BUT 5층 보스는 테마 벗어날 수 있음 → 방심 불가
- "오늘 화산이네, 갸라도스로 가야지" → 매일 다른 전략

## 4. 적 생성 & 난이도 스케일링

### 4-1. 적 포켓몬 생성
- 테마 타입 풀에서 60% 확률, 기타 타입에서 40% 확률로 랜덤 선택
- 적 희귀도: 층수에 비례 (1~5층 커먼, 6~15층 레어 혼합, 16+ 에픽 혼합)
- 적 IV: 0 고정 (순수 스케일링으로 난이도 조절)

### 4-2. 난이도 스케일링 (구간별 가속)

```python
def enemy_scaling(floor):
    if floor <= 10:
        return 1.0 + floor * 0.06       # 1~10: 1.06~1.60 (입문)
    elif floor <= 20:
        return 1.60 + (floor-10) * 0.10  # 11~20: 1.70~2.60 (중반)
    elif floor <= 30:
        return 2.60 + (floor-20) * 0.15  # 21~30: 2.75~4.10 (후반)
    else:
        return 4.10 + (floor-30) * 0.20  # 31+: 4.30~∞ (극한)
```

| 층 | 배율 | 체감 |
|----|------|------|
| 5 | ×1.30 | 쉬움 — 누구나 클리어 |
| 10 | ×1.60 | 보통 — 초보 벽 |
| 15 | ×2.10 | 어려움 — 미육성 포켓몬 한계 |
| 20 | ×2.60 | 하드 — 이로치/A급 IV 필요 |
| 25 | ×3.35 | 극하드 — S급 + 버프 빌드 필요 |
| 30 | ×4.10 | 극한 — 풀투자 + 완벽 빌드 |
| 40 | ×6.10 | 초극한 — 이론상 한계 |
| 50 | ×8.10 | 전설 — 퍼펙트 런 전용 |

### 4-3. 층 타입
| 타입 | 확률 | 설명 |
|------|------|------|
| 일반 | 60% | 적 1마리 |
| 엘리트 | 20% | 적 스탯 ×1.3, 보상 ×1.5 |
| 이벤트 | 10% | 특수 선택 (상점/도박/보물) |
| 보스 | 고정 | 5/10/15/20... 층마다 |

### 4-4. 이벤트 층
```
🏪 상점: BP 소모해 HP 30% 회복 (50BP) 또는 버프 1개 구매 (100BP)
🎁 보물: 무조건 버프 1개 추가 획득
❓ 도박: HP 20% 걸기 → 성공(50%): 희귀 버프, 실패: HP 감소
🔄 변환: 현재 버프 1개를 한 등급 위로 업그레이드
```

## 5. 버프 시스템

### 5-1. 코스트별 버프 주기

| 희귀도 | 코스트 | 버프 주기 | 근거 |
|--------|--------|----------|------|
| 커먼 | 1 | **매 층** | 버프 잔뜩 쌓아서 역전 |
| 레어 | 2 | **매 층** | 커먼과 동급 |
| 에픽 | 4 | **2층마다** | 기본 스탯으로 중반 커버 |
| 전설 | 5 | **3층마다** | 높은 기본 스탯 |
| 초전설 | 6 | **4층마다** | 압도적 기본 스탯, 버프 최소 |

→ 20층 기준: 커먼 버프 20개 vs 초전설 버프 5개
→ 커먼은 버프 시너지 빌드, 초전설은 스탯 압살

- 시너지 조합은 **미공개** → 발견하는 재미
- 버프 설명에 힌트: "화염 강타 — 불꽃 계열과 함께하면..."
- 시너지 발견 시 **던전 도감**에 등록

## 6. 상성 시스템 (던전용)

기본 배틀과 동일하되, 면역만 완화:

| 상성 | 배율 | 비고 |
|------|------|------|
| 유리 | ×1.5 | 기존과 동일 |
| 불리 | ×0.67 | 기존과 동일 |
| 면역 | ×0.3 | **던전 전용 완화** (기존 ×0) |

→ 면역 타입이 나와도 "어렵지만 불가능하진 않음"
→ 상성 버프로 극복 가능 = 버프 선택의 전략적 가치 ↑

## 8. 보상 체계

### 8-1. 런 보상 (종료 시 일괄 지급)

```
층수 × 10 BP (기본)
+ 보스 클리어 보너스
+ 이벤트 층 보너스
```

| 마일스톤 | 보상 |
|---------|------|
| 5층 보스 | +50 BP, 테마 조각 1개 |
| 10층 관장 | +100 BP, 테마 조각 2개, 입장권 1장 |
| 15층 보스 | +150 BP, 테마 조각 2개 |
| 20층 관장 | +200 BP, 테마 조각 3개, 결정 1개 |
| 25층 보스 | +250 BP, 결정 1개 |
| 30층 관장 | +300 BP, 결정 2개, 무지개 결정 1개 |
| 40층 관장 | +400 BP, 결정 3개, 무지개 결정 1개, IV스톤 1개 |
| 50층 관장 | +500 BP, 무지개 결정 2개, IV스톤 1개, 칭호 |

**테마 조각**: 당일 던전 테마에 해당하는 캠프 조각 (화산 → 화산 조각)
→ 던전 돌면 캠프 조각이 쌓임 → 이로치 전환 → 더 강한 포켓몬 → 더 높은 층

### 8-2. 신규 아이템: IV 스톤

| 아이템 | 효과 | 획득처 |
|--------|------|--------|
| IV 스톤 | 선택한 IV 스탯 +3 (최대 31) | 던전 40층+, 시즌 보상 |

→ 가챠에서만 나오던 IV 리롤의 **확정 상위호환**
→ 던전 고층 도달의 핵심 동기
→ 기존 경제와 충돌 없음 (리롤은 랜덤, 스톤은 확정 +3)

### 8-3. 신규 아이템: 만능 조각

| 아이템 | 효과 | 획득처 |
|--------|------|--------|
| 만능 조각 | 아무 필드 타입의 조각으로 사용 가능 | 던전 20층+, 시즌 보상 |

→ "숲 조각은 넘치는데 화산 조각이 없다" 문제 해결
→ 캠프 시스템 활성화 촉진

## 9. 포켓몬 스탯 반영

던전에서의 내 포켓몬 전투력:

```
기본 배틀 스탯 그대로 사용 (기존 calc_battle_stats)
= 종족값 × 성격배율 × (1 + 친밀도×0.04) × 진화단계 × IV보정 × 희귀도보정
```

투자에 따른 차이:
| 상태 | 실효 배율 | 예상 한계층 |
|------|----------|-----------|
| 커먼 C급 일반 | ×1.0 (기준) | ~10층 |
| 레어 A급 일반 | ×1.15 | ~15층 |
| 에픽 A급 이로치 | ×1.55 | ~22층 (버프 2층마다) |
| 전설 S급 이로치 | ×1.75 | ~28층 (버프 3층마다) |
| 커먼 S급 이로치 | ×1.36 + 매층 버프 | ~35층+ (버프 빌드) |
| 초전설 S급 이로치 | ×2.0 | ~32층 (버프 4층마다) |

→ **커먼 S급 이로치 + 시너지 빌드 > 초전설 기본**이 가능한 구조

## 11. 보상 루프 요약

```
[일일 루프]
던전 도전 → BP + 테마 조각 획득
         → 조각으로 이로치 전환 (캠프)
         → 이로치 포켓몬으로 더 높은 층
         → 더 많은 보상

[주간 루프]
주간 랭킹 → 결정/입장권/마볼 보상
       → 결정으로 에픽+ 이로치 전환
       → 전투력 상승 → 다음 주 더 높은 순위

[시즌 루프]
시즌 마일스톤 → IV 스톤/만능 조각
            → IV 최적화 → 영구 전투력 상승
            → 다음 시즌 더 높은 기록

[육성 루프]
포획 (좋은 IV) → 밥/놀기 (친밀도) → 캠프 (조각)
→ 이로치 전환 → 던전 도전 → 보상 → 더 나은 포켓몬 육성
```

## 12. DB 스키마

```sql
-- 유저 던전 관련 컬럼
ALTER TABLE users ADD COLUMN dungeon_tickets INT NOT NULL DEFAULT 1;
ALTER TABLE users ADD COLUMN dungeon_tickets_bought_today INT NOT NULL DEFAULT 0;
ALTER TABLE users ADD COLUMN dungeon_best_floor INT NOT NULL DEFAULT 0;
ALTER TABLE users ADD COLUMN dungeon_season_runs INT NOT NULL DEFAULT 0;

-- 던전 런 기록
CREATE TABLE dungeon_runs (
    id SERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    pokemon_instance_id INT NOT NULL,
    pokemon_id INT NOT NULL,
    pokemon_name TEXT NOT NULL,
    is_shiny BOOLEAN DEFAULT FALSE,
    iv_grade TEXT,
    floor_reached INT NOT NULL DEFAULT 0,
    theme TEXT NOT NULL,
    buffs_json JSONB DEFAULT '[]',
    synergies_found JSONB DEFAULT '[]',
    bp_earned INT DEFAULT 0,
    fragments_earned INT DEFAULT 0,
    started_at TIMESTAMPTZ DEFAULT NOW(),
    ended_at TIMESTAMPTZ,
    season_key TEXT
);

CREATE INDEX idx_dungeon_ranking ON dungeon_runs(season_key, floor_reached DESC);
CREATE INDEX idx_dungeon_user ON dungeon_runs(user_id, started_at DESC);

-- 포켓몬별 던전 최고 기록
CREATE TABLE dungeon_pokemon_records (
    user_id BIGINT NOT NULL,
    pokemon_instance_id INT NOT NULL,
    best_floor INT NOT NULL DEFAULT 0,
    best_theme TEXT,
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    PRIMARY KEY (user_id, pokemon_instance_id)
);

-- 시너지 도감 (발견한 조합)
CREATE TABLE dungeon_synergy_log (
    user_id BIGINT NOT NULL,
    synergy_key TEXT NOT NULL,
    discovered_at TIMESTAMPTZ DEFAULT NOW(),
    PRIMARY KEY (user_id, synergy_key)
);

-- 신규 아이템: IV 스톤, 만능 조각
ALTER TABLE users ADD COLUMN iv_stones INT NOT NULL DEFAULT 0;
ALTER TABLE users ADD COLUMN universal_fragments INT NOT NULL DEFAULT 0;
```
