## 1. Clash of Clans

### 1-1. 클랜전 (Clan War) 핵심 메카닉

**구조: 준비일(1일) + 전투일(1일)**
- 매칭: 클랜 리더가 수동으로 참전 멤버 선발 (15v15, 30v30 등)
- 매칭 알고리즘: 참전 멤버들의 전투력 합산으로 상대 클랜 탐색. 타운홀 레벨, 방어시설, 유닛 레벨 등 가중치 계산
- 준비일: 상대 기지 정찰 가능. 전략 논의. 워 베이스 배치 변경
- 전투일: 각 멤버 2회 공격 기회. 별 1~3개 획득 (파괴율 기반)
- 승리 조건: 총 별 수 비교 → 동률 시 파괴율

**자원 경제:**
- 승리 시 워 보너스(골드, 엘릭서) → 개인에게 분배
- 패배해도 소량 보상 존재 (참여 동기 유지)
- 전쟁 자체에 입장 비용 없음 (진입 장벽 최소화)

**미스매치 방지:**
- 엔지니어드 베이스(의도적 저레벨 방어) 대응: 공격력+방어력 모두 매칭에 반영
- 여러 차례 알고리즘 수정을 거침 (초기에는 방어 무게만 높아서 어뷰징 심했음)
- 매칭 풀이 부족하면 대기 시간이 늘어나되, 미스매치 허용 범위를 점진적으로 넓힘

### 1-2. 클랜전 리그 (CWL)

**구조: 월간 시즌, 7일간 라운드 로빈**
- 8개 클랜이 한 그룹 → 7일간 매일 1전씩 총 7전
- 각 리그 티어 존재 (챔피언, 마스터, 크리스탈, 골드, 실버, 브론즈)
- 시즌 결과로 승격/잔류/강등 결정
- CWL 메달 보상 → 특별 아이템 구매

**핵심 설계 포인트:**
- 라운드 로빈이라 1패해도 시즌 전체가 망하지 않음
- 승격/강등으로 장기적 경쟁 동기
- 참전 인원 고정(15명 등)으로 소규모 클랜도 참여 가능

### 1-3. 클랜 캐피탈 (Clan Capital)

- 클랜원 모두가 자원을 기부하여 공동 거점 건설
- 주말마다 레이드 위크엔드 → 다른 클랜 캐피탈 공격
- 개인이 아닌 클랜 전체가 함께 건설하는 협동 콘텐츠
- 레이드 메달로 개인 보상

### 1-4. 재미/중독 요소

1. **준비 + 실행 분리**: 전략 수립 시간이 있어서 소통 촉진
2. **2회 공격 제한**: 긴장감. 실수하면 클랜에 피해
3. **사회적 압력/보상**: 잘하면 칭찬, 못하면 자연스러운 동기부여
4. **비동기 가능**: 24시간 안에 공격하면 됨. 실시간 접속 불필요
5. **적당한 빈도**: 주 1~2회 전쟁으로 번아웃 방지

### 1-5. 문제점과 해결

| 문제 | 해결 |
|------|------|
| 엔지니어드 베이스 어뷰징 | 공격력도 매칭에 반영하도록 알고리즘 수정 |
| 미참여 멤버 (공격 안 함) | 리더에게 참전 멤버 선발 권한 부여 |
| 미스매치 불만 | 매칭 알고리즘 지속 개선 + CWL 리그제 도입 |
| 과금 격차 | CWL에서 티어 분리로 동급끼리 경쟁 |

### 1-6. TGPoke 적용 가능성

- **준비일 + 전투일 구조**: 캠프전에서 "전략 수립 시간"과 "실행 시간" 분리 가능
- **별 시스템(1~3)**: 포켓몬 배치 점수를 단순화한 승패 판정
- **비동기 공격**: 텔레그램 봇에 완벽히 적합. 기한 내 DM으로 공격
- **월간 리그 + 승강제**: 캠프 간 시즌제 랭킹으로 직접 적용 가능
- **CWL 메달 = BP 또는 새 보상 화폐**: 시즌 보상으로 참여 동기

## 2. Lineage / Lineage 2

### 2-1. 공성전 (Castle Siege)

**구조: 주 1회, 정해진 시간에 실시간 진행 (2시간)**
- 성을 보유한 혈맹(클랜)이 방어, 도전 혈맹이 공격
- 성문 파괴 → 내성 진입 → 왕좌 점령 → 일정 시간 유지 시 승리
- 성 보유 혈맹: 세금 수입, NPC 상점 통제, 특수 버프

**영토전:**
- 성 주변 영지 통제 → 사냥터 세금, 자원 보너스
- 혈맹 간 동맹 시스템으로 대규모 연합전 가능
- 영토마다 다른 보너스 (사냥터, 광산, 항구 등)

### 2-2. 혈맹 구조

- 혈맹주 → 부혈맹주 → 근위대장 → 일반
- 혈맹 레벨 시스템: 멤버 활동으로 혈맹 포인트 축적
- 혈맹 스킬: 레벨 올리면 전체 멤버에게 버프 (공격력, 경험치 등)
- 혈맹원 활동량이 혈맹 성장에 직접 기여

### 2-3. 웨일 vs F2P 밸런스

**문제:**
- 초기 리니지2는 극심한 과금 격차. 고레벨 장비 = 현금
- 소수 웨일이 성을 독점, F2P 유저 이탈

**대응:**
- 공성 참여 인원 제한 (동시 접속 인원 캡)
- 성벽/성문이라는 지형 이점으로 방어 측 유리 (숫자 불리 보완)
- 동맹 시스템으로 약한 혈맹도 연합하여 대항 가능
- 시간 제한(2시간)으로 무한 소모전 방지
- 후속작(리니지W 등)에서는 시즌제 도입하여 리셋

### 2-4. 어뷰징 방지

- 혈맹 이적 쿨다운 (탈퇴 후 일정 기간 가입 불가)
- 공성 참여 자격: 혈맹 가입 후 최소 기간 필요
- 동맹 등록도 공성 전 일정 기간 전에 확정해야 함
- 캐릭터 레벨/장비 요구사항으로 계정 파밍 제한

### 2-5. 재미/중독 요소

1. **정치적 요소**: 동맹, 배신, 외교 → 커뮤니티 드라마
2. **실질적 보상**: 성 세금 수입은 진짜 게임 내 경제에 영향
3. **명예**: 서버 내 최강 혈맹이라는 사회적 지위
4. **주 1회 이벤트**: 한 주 동안 준비하고 주말에 결전
5. **역전 가능성**: 동맹 재편으로 판도 변화

### 2-6. TGPoke 적용 가능성

- **세금/수입 시스템**: 캠프가 영토를 보유하면 BP 수입 발생
- **동맹 시스템**: 캠프 간 동맹으로 연합전 가능성
- **정치/외교**: 텔레그램 채팅방이라는 환경에 매우 적합 (자연스러운 소통)
- **쿨다운/이적 제한**: 이미 거점 변경 7일 쿨다운이 있으므로 확장 가능
- **지형 이점**: 방어 측에 약간의 보너스를 주어 영구 독점 방지

## 3. Guild Wars 2

### 3-1. World vs World (WvW)

**구조: 3서버 대전, 주간 매치업**
- 3개 서버가 4개 맵에서 동시 전투
- 거점(성, 탑, 캠프)을 점령하고 방어
- 5분마다 점수 틱: 보유 거점 수에 비례하여 점수 획득
- 주간 총점으로 순위 결정 → 다음 주 매치업 변경

**영토 통제:**
- 각 거점은 보급품(Supply) 필요 → 성벽 건설, 방어 시설 업그레이드
- Supply는 캠프(작은 거점)에서 생산 → 큰 성으로 운반
- 보급선 차단 전략이 핵심

### 3-2. Supply/자원 관리

- Supply Camp → Tower → Keep → Castle 순으로 크기/중요도
- 작은 캠프를 빼앗으면 적 성의 보급 차단
- 성벽 업그레이드, 공성 무기 설치에 Supply 소모
- 전략적 자원 관리가 단순 전투력 이상으로 중요

### 3-3. 스코어링

- 5분마다 PPT (Points Per Tick) 계산
- 보유 거점 수 × 거점 가치 = 틱당 점수
- 1주일간 축적 → 최종 순위
- 보너스 이벤트 (야간 보너스 등)로 시간대별 참여 유도

### 3-4. 문제점과 해결

| 문제 | 해결 |
|------|------|
| 인구 불균형 (한 서버만 강함) | 서버 연결(linking) 시스템으로 약한 서버 합치기 |
| 야간 기습 (비활성 시간대 점령) | 야간 보너스 점수 감소, 거점 자동 방어 NPC |
| 신규 유저 진입 장벽 | 레벨 스케일링 (낮은 레벨도 기여 가능) |
| 한쪽이 압도적으로 이기면 참여율 저하 | 주간 리셋 + 매치업 재편 |

### 3-5. 재미/중독 요소

1. **지속적 점수 축적**: 한 번의 전투가 아닌 1주일간의 영토 유지
2. **보급선 전략**: 단순 전투력이 아닌 전략적 사고 필요
3. **규모의 다양성**: 소규모 유격대, 대규모 정면 충돌 모두 가치
4. **24시간 진행**: 아무 때나 참여 가능
5. **주간 리셋**: 매주 새로운 시작, 패배해도 다음 주 기회

### 3-6. TGPoke 적용 가능성

- **PPT (Points Per Tick) 시스템**: 현재 캠프의 3시간 정산과 매우 유사! 거점 보유 시 틱당 점수 자연 적용 가능
- **Supply 개념**: 캠프 조각을 Supply처럼 활용 → 방어 업그레이드에 소모
- **주간 리셋**: 시즌제와 결합하여 매주 영토 경쟁
- **3팀 대전**: 2팀이 아닌 3팀 이상 경쟁으로 동맹/견제 역학
- **소규모 기여 가치**: 캠프 배치(소규모 활동)가 전체 점수에 기여

## 4. Clash Royale

### 4-1. 클랜전쟁 2 (Clan Wars 2) — River Race

**구조: 주간 강 레이스**
- 5개 클랜이 한 레이스에 배정
- 각 클랜원이 매일 전투(PvP, 듀얼, 보트전)로 명예 포인트 획득
- 클랜 합산 명예로 배(Boat)를 전진시킴
- 결승선에 먼저 도달한 클랜이 우승

**개인 기여:**
- 1일 4전 제한 (4개 덱 사용, 각 1회)
- 덱 쿨다운: 사용한 카드는 당일 재사용 불가 → 다양한 덱 필요
- PvP 승리 > 듀얼 승리 > 보트전 순으로 명예 포인트 차등

**보트 메카닉:**
- 적 클랜의 보트를 공격하여 지연시킬 수 있음
- 보트가 침몰하면 수리에 시간 소모 → 전략적 방해

### 4-2. 문제점과 해결

| 문제 | 해결 (또는 미해결) |
|------|------|
| 소규모 클랜 불리 | 멤버 수에 비례한 보정은 부분적으로만 적용 → 여전히 문제 |
| 카드 레벨 격차 | 듀얼 모드에서 토너먼트 레벨 적용 (일부) |
| 비참여 멤버 | 최소 참여 인원 미달 시 페널티 |
| 월~목은 무의미 (주말 스퍼트) | 매일 참여하면 보너스 명예 |

### 4-3. 재미/중독 요소

1. **매일 조금씩 기여**: 4전이면 10~15분, 부담 적음
2. **레이스 시각화**: 배가 전진하는 것을 실시간으로 보는 재미
3. **방해 전략**: 1등 견제 가능 → 순위 역전 드라마
4. **다양한 전투 모드**: PvP, 듀얼, 보트전 선택지

### 4-4. TGPoke 적용 가능성

- **일일 참여 제한(4회)**: 캠프전에서도 일일 공격 횟수 제한으로 적용
- **레이스/경주 형식**: 캠프 간 주간 포인트 경주 → 직관적이고 시각화 쉬움
- **방해 메카닉**: 상대 캠프 방해(포인트 깎기 등) 가능
- **매일 소량 참여**: 텔레그램 봇 사용 패턴에 완벽히 부합
- **다양한 기여 방식**: 배치 뿐 아니라 배틀, 포획 등 다양한 방법으로 캠프전 점수 기여

## 5. Rise of Kingdoms / Lords Mobile

### 5-1. 영토/맵 기반 전쟁

**Rise of Kingdoms:**
- 실제 월드맵에서 동맹 영토 확장
- 자원 타일(농장, 광산, 벌목장) 점령 → 자원 생산
- 성지(Holy Sites), 로스트 킹덤(Lost Kingdom) 등 특별 영토 쟁탈
- KvK (Kingdom vs Kingdom): 서버 간 대규모 전쟁

**Lords Mobile:**
- 원더(Wonder) 점령전: 맵 중앙 거점 차지 시 왕국 통치권
- 자원 타일 파밍 → 적 동맹 타일 약탈 가능
- 다크네스트(PvE 협동) → 길드 선물 보상

### 5-2. 동맹/얼라이언스 메카닉

- 동맹 기술 연구: 동맹원 기부로 공동 기술 해금 (건설 속도, 전투력 보너스 등)
- 동맹 상점: 기부 포인트로 아이템 구매
- 동맹 영토: 깃발 설치 → 연결된 영역이 동맹 영토
- 영토 내 이동 속도 보너스, 자원 보호 등 혜택

### 5-3. 자원 노드와 통제

- 각 자원 타일에 유한한 자원량 → 고갈되면 이동 필요
- 고급 자원 타일은 맵 중앙(위험 지역)에 위치 → 리스크/리워드
- 동맹이 자원 지역을 영토로 확보 → 안전 파밍
- 적 파밍 유저 공격으로 자원 약탈 가능

### 5-4. 마이그레이션 (서버 이전)

- 유저가 다른 서버로 이주 가능 (아이템 필요)
- 강한 유저가 약한 서버로 이주하여 지배하는 문제
- 대응: 이주 시 전투력 상한, 신규 서버 이주 제한, 이주 쿨다운

### 5-5. 문제점과 해결

| 문제 | 해결 |
|------|------|
| 웨일 동맹 독점 | KvK에서 서버 통합으로 경쟁 상대 제공 |
| 비활성 유저 영토 | 일정 기간 미접속 시 영토 자동 해제 |
| 신규 유저 약탈 | 초보자 보호막(쉴드), 동맹 구역 내 보호 |
| 독재적 동맹주 | 탈퇴 자유, 동맹 간 이적 활발 |

### 5-6. TGPoke 적용 가능성

- **영토 맵 시각화**: 캠프 간 영토를 단순화된 맵으로 표현 가능
- **자원 노드**: 특정 포인트를 캠프가 점령 → 추가 BP 수입, 특별 스폰 등
- **동맹 기술/공동 연구**: 캠프 레벨업과 유사하게 확장 가능
- **마이그레이션 제한**: 이미 거점 변경 7일 쿨다운으로 기본 프레임 있음

## 6. Pokemon GO

## 7. Anti-Abuse 시스템 종합

### 7-1. 멀티 계정 탐지

**일반적 방법:**
| 방법 | 적용 가능성 (텔레그램) | 비고 |
|------|:---:|------|
| IP 주소 추적 | ❌ | 텔레그램 봇은 유저 IP 접근 불가 |
| 디바이스 핑거프린팅 | ❌ | 웹 앱이 아니므로 불가 |
| 전화번호 연결 | ⚠️ | 텔레그램 가상번호 가입 쉬움 |
| 행동 패턴 분석 | ✅ | 가장 현실적인 방법 |
| 소셜 그래프 분석 | ✅ | 같은 채팅방, 교환 패턴 분석 |

**텔레그램 봇에서 가능한 행동 패턴 분석:**
- 동일 시간대 활동 패턴 (같은 사람이 여러 계정 운영)
- 일방적 교환 패턴 (A→B로만 좋은 포켓몬 전송)
- 동일 채팅방에서 동시 활동하는 계정 쌍
- 포획/배틀 타이밍 패턴 유사도
- 하나의 계정이 항상 다른 특정 계정과만 상호작용

### 7-2. 봇(자동화) 탐지

**일반 게임 대응:**
- CAPTCHA (정기적/의심 시)
- 행동 분석: 클릭 간격, 반응 속도, 패턴 규칙성
- 하니팟: 일반 유저는 무시하지만 봇은 반응하는 트리거

**텔레그램 봇 맥락:**
- 응답 속도 분석 (인간 불가능한 속도로 일관적 반응)
- 24시간 무중단 활동 감지
- 명령어 입력 패턴: 봇은 오타 없이 완벽한 명령만 입력
- 간헐적 인터렉티브 확인 (인라인 버튼 선택 요구)

### 7-3. 자원 파이프라인(Funneling) 방지

**문제:** 다수 계정으로 자원 수집 → 메인 계정에 집중

**대응 (타 게임):**
- 교환 일일 제한 (이미 TGPoke에 10회 제한 적용)
- 교환 세금 (BP 또는 포켓몬 가치의 일정 % 소멸)
- 신규 계정 교환 제한 (가입 후 N일 이후 교환 가능)
- 교환 가치 불균형 경고 (한쪽이 압도적으로 비싼 포켓몬 보냄)
- 주간/월간 교환 총량 상한

### 7-4. 교환 어뷰징 방지

**클래시오브클랜 접근:**
- 기부 쿨다운
- 타운홀 레벨 기반 요청 제한

**포켓몬GO 접근:**
- 교환 시 IV 재추첨 (고IV 보장 교환 불가)
- 특별 교환 일일 1회 제한
- 친밀도 기반 교환 비용 (별의 모래)

**TGPoke 적용:**
- 교환 시 IV 변동 (±1~2 범위 재추첨) → 완벽한 포켓몬 교환 리스크
- 교환 수수료 (BP)
- 계정 나이 기반 교환 등급 (7일 미만 → 제한적 교환)

### 7-5. 신고 시스템

**모범 사례:**
- 신고 카테고리 명확화 (멀티어카, 봇, 어뷰징, 비매너 등)
- 신고 누적 시 자동 제재 (소프트 → 하드)
- 오남용 방지: 허위 신고 페널티
- 관리자 검토 대기열

**TGPoke 적용:**
- `/report` 명령으로 신고
- 채팅방 관리자(채널장)에게 기본 조사 권한
- 관리자(문유)에게 최종 제재 권한
- 캠프전 관련 어뷰징은 자동 감지 + 관리자 확인 이중 구조

### 7-6. 캠프전 특화 어뷰징 시나리오

| 시나리오 | 탐지 | 대응 |
|----------|------|------|
| 부계정으로 상대 캠프 가입 → 고의 패배 | 일방적 패배 패턴, 신규 계정 | 캠프전 참여 자격: 가입 N일 이상 |
| 부계정으로 자기 캠프 점수 올리기 | 교환/활동 패턴 분석 | 캠프전 기여 최소 요구사항 |
| 캠프 간 담합 (순위 조작) | 비정상적 점수 패턴 | 랜덤 매칭, 라운드 로빈 |
| 캠프전 직전 강한 유저 영입 | 가입 시점 확인 | 캠프전 참여: 거점 설정 N일 이상 |

## 8. 크로스커팅 분석

### 8-1. F2P vs 웨일 밸런스

**핵심 원칙들 (전 게임 공통):**

1. **기여도 상한 (Cap)**: 개인이 기여할 수 있는 양에 상한을 둠
   - CoC: 2회 공격 제한
   - CR: 4전 제한
   - PoGO: 일일 50코인
   - → TGPoke: 일일 배치 횟수 제한 (이미 존재)

2. **티어/리그 분리**: 비슷한 수준끼리 경쟁
   - CoC CWL: 챔피언~브론즈 리그
   - → TGPoke: 캠프 레벨 기반 리그 분리

3. **숫자의 힘**: 개인 과금보다 참여 인원이 중요하게 설계
   - GW2: 대규모 전투에서 개인 전투력보다 인원수
   - RoK: 동맹 인원수가 영토 유지에 핵심
   - → TGPoke: 채팅방 활성 멤버 수가 캠프전 핵심 변수

4. **다양한 기여 경로**: 전투 외 방법으로도 기여
   - CoC: 기지 방어도 중요
   - GW2: 보급 운반, 건설
   - → TGPoke: 배치, 배틀, 포획, 교환 등 다양한 활동이 캠프전 점수에 기여

### 8-2. 시즌/주기적 시스템

**모범 사례:**

| 게임 | 주기 | 리셋 범위 | 보상 |
|------|------|-----------|------|
| CoC CWL | 월간 | 리그 순위만 | CWL 메달 |
| GW2 WvW | 주간 | 영토 전체 | 스카미시 티켓 |
| CR 클랜전 | 주간 | 레이스 위치 | 전쟁 상자 |
| RoK KvK | 계절 | 별도 맵 | KvK 보상 |
| PoGO 시즌 | 3개월 | 랭크 | 시즌 보상 |

**TGPoke 권장:**
- **주간**: 캠프전 한 사이클 (월~일)
- **월간**: 시즌 결산, 리그 승강
- **분기**: 메타 변경, 특별 이벤트

### 8-3. 영토의 의미 (비실시간 맥락)

**텔레그램 봇에서 "영토"를 의미있게 만드는 방법:**

1. **수동적 수입**: 영토 보유 = 자동 BP/조각 수입 (PoGO 방식)
2. **특수 스폰**: 특정 영토 보유 시 해당 채팅방에서 레어 포켓몬 스폰율 증가
3. **버프 효과**: 영토 보유 캠프원에게 배틀 버프, 포획률 보너스 등
4. **명예/가시성**: 대시보드에서 영토 보유 캠프 표시 → 자랑
5. **제한된 공급**: 영토 수를 캠프 수보다 적게 → 경쟁 유발

### 8-4. 안티 스노볼 (Anti-Snowball)

**강한 캠프가 계속 강해지는 것을 방지하는 메카닉들:**

1. **수확 체감 (Diminishing Returns)**
   - 영토를 많이 가질수록 방어 비용 증가
   - GW2: 많은 거점 보유 시 Supply 분산으로 방어 약화
   - → TGPoke: 영토 N개 이상 시 유지 비용(조각/BP) 급증

2. **방어자 소모 (Defender Fatigue)**
   - 배치한 포켓몬의 CP/체력이 시간에 따라 감소 (PoGO)
   - → TGPoke: 연속 방어 시 점수 감소 보정

3. **도전자 보너스 (Challenger Advantage)**
   - 하위 캠프가 상위 캠프를 이기면 추가 보상
   - 다윗 vs 골리앗 구조

4. **주기적 리셋**
   - 시즌마다 영토 리셋 → 재경쟁
   - 리그 시스템으로 자연 밸런싱

5. **자원 분산 메카닉**
   - 1등 캠프에게 불리한 매칭
   - 하위 캠프에게 캐치업 보너스 (경험치, BP 추가 등)

6. **인원 기반 밸런싱**
   - 멤버 수 대비 기여도 계산 (인당 효율)
   - 대규모 캠프와 소규모 캠프 공정 경쟁

## 9. TGPoke 적용 종합 제안

### 9-1. 캠프전 기본 구조 (제안)

현재 시스템(3시간 정산, 상시 배치, 거점캠프)을 기반으로 확장:

```
┌─────────────────────────────────────┐
│          캠프전 주간 사이클           │
│                                     │
│  월  화  수  목  금  [토  일]       │
│  준비────────→전투──→│정산│         │
│                                     │
│  월~금: 평소 캠프 활동 (배치, 정산)  │
│  + 캠프전 포인트 축적               │
│  토~일: 캠프 vs 캠프 대결           │
│  일 24:00: 주간 결과 정산           │
└─────────────────────────────────────┘
```

### 9-2. 매칭 시스템

**리그제 (CoC CWL + GW2 참고):**
- 캠프 레벨 + 활성 멤버 수 기반으로 리그 배정
- 리그 내 4~8개 캠프가 한 그룹
- 주간 라운드 로빈 또는 포인트 경주
- 시즌(월간) 종료 시 승격/강등

**리그 구분 예시:**
| 리그 | 조건 | 매칭 |
|------|------|------|
| 브론즈 | 캠프 Lv.1~3 | 4캠프 1그룹 |
| 실버 | 캠프 Lv.4~6 | 4캠프 1그룹 |
| 골드 | 캠프 Lv.7~8 | 4캠프 1그룹 |
| 마스터 | 캠프 Lv.9~10 | 4캠프 1그룹 |

### 9-3. 캠프전 점수 기여 방식 (CR + GW2 참고)

**다양한 활동이 캠프전 포인트에 기여:**

| 활동 | 캠프전 포인트 | 일일 상한 |
|------|:---:|:---:|
| 캠프 배치 정산 | 배치 점수의 100% | 기존 배치 제한 |
| 랭크 배틀 승리 | +3 | 10전 |
| 일반 배틀 승리 | +1 | 10전 |
| 포켓몬 포획 | +1 | 20회 |
| 캠프전 대결 (직접 공격) | +5~15 | 3회 |

### 9-4. 캠프전 대결 메카닉 (CoC 참고)

**비동기 배틀 시스템:**
1. 상대 캠프의 "방어 라인업" 공개 (배치된 포켓몬 목록)
2. 공격 캠프원이 자기 포켓몬으로 도전 (DM에서 비동기)
3. 자동 배틀 → 결과 표시 (별 1~3개)
4. 1인 일일 3회 공격 제한
5. 같은 방어 라인업 중복 공격 불가 (다른 대상 공격해야 함)

### 9-5. 영토 시스템 (RoK + GW2 참고)

- 캠프전 승리 시 영토 선택/빼앗기
- 영토 보유 효과는 캠프원 전체에 적용

### 9-6. 안티 어뷰징 (캠프전 특화)

1. **캠프전 참여 자격**: 거점 설정 후 3일 이상 경과
2. **최소 기여**: 주간 최소 N포인트 기여해야 보상 수령
3. **멤버 수 보정**: 캠프전 점수 = 총점 ÷ √(참여인원) → 소규모 캠프 보호
4. **교환 제한 강화**: 캠프전 기간 중 캠프 간 교환 추가 제한
5. **부계정 감시**: 캠프전 직전 가입 계정 자동 플래그

### 9-7. 보상 체계

### 9-8. 구현 우선순위 제안

## 부록: 핵심 교훈 요약

### 전 게임 공통으로 검증된 설계 원칙

1. **비동기 참여가 핵심**: 모든 성공한 길드전은 기한 내 자유 참여 (실시간 강제 X)
2. **개인 기여 상한**: 웨일 1명이 캐리하는 것 방지 → 참여 인원이 중요
3. **최소 보상 보장**: 져도 뭔가 받아야 다음에도 참여
4. **주기적 리셋**: 영구 독점 방지, 새로운 기회 제공
5. **다양한 기여 방식**: 전투 외에도 기여할 방법이 있어야 캐주얼 유저 유지
6. **시각적 진행**: 점수판, 레이스, 영토 맵 등 진행 상황의 가시화
7. **사회적 압력/유대**: 길드원에게 피해주기 싫은 마음 → 자발적 참여
8. **준비 + 실행 분리**: 전략 수립 시간이 커뮤니케이션을 촉진

### 텔레그램 봇 특수 고려사항

- 실시간 그래픽 없음 → 텍스트/이미지 기반 시각화 (카드 이미지, 대시보드)
- 알림 = DM → 정산/결과를 DM으로 자연스럽게 전달 (이미 구축됨)
- 채팅방 = 길드홀 → 별도 길드 시스템 불필요 (기존 채팅방이 곧 캠프)
- 인라인 버튼 = UI → 복잡한 조작도 버튼으로 단순화 가능
- 봇 명령어 = 비동기 액션 → 24시간 아무 때나 공격/배치 가능
